# 钓鱼网站识别

> **技能补给站**
> **本课目标**：掌握钓鱼网站的 5 大特征和 3 步验证法，遇到可疑链接/邮件/网站时能冷静判断，不被骗。

## 这节课解决什么问题

上一课学了怎么看域名，这一课学怎么识破**整条诈骗链**。钓鱼攻击不会只给你一个网址让你判断——它会包装成一封看起来很正规的邮件、一条短信、一个弹窗，用紧迫感和恐惧感逼你点击。

亚马逊运营是钓鱼攻击的高频目标，因为你手里有：**卖家账号（能提现）、买家付款数据、供应商货款**。一旦被骗，损失的不只是自己，还有客户和公司。这节课教你识别完整的钓鱼套路。

## 核心知识

### 钓鱼攻击的 5 个特征

#### 特征 1：制造紧迫感

这是钓鱼邮件最典型的手法。它要让你**来不及思考就点击**。

| 正规通知 | 钓鱼通知 |
|---|---|
| 「您的账户有新消息，请登录查看」 | 「⚠️ 您的账号将在 24 小时内被永久封禁！立即验证！」 |
| 「请及时更新税务信息」 | 「🚨 紧急：您的资金已被冻结！点击解冻！」 |

> **金句：任何催你「立即」「马上」「24小时内」行动的，先停下来。正规平台不会用恐吓逼你操作。**

#### 特征 2：域名造假（上一课的内容）

钓鱼网站的域名套路：

```
真：sellercentral.amazon.com
假套路：
  ❌ amazon.verify-secure.com     （主域名不是 amazon）
  ❌ sellercentral.amazon.com.fake.net  （主域名是 fake）
  ❌ sellerceniral.amazon.com      （l 和 r 互换，肉眼难辨）
  ❌ amaz0n.com                     （数字 0 代替字母 o）
  ❌ www.amazon.com.secure-login.xyz （长域名混淆）
```

#### 特征 3：要求输入敏感信息

正规亚马逊**绝不会通过邮件链接**要求你输入：
- 密码
- 信用卡完整号码 + CVV
- 验证码 / 2FA 验证码
- 银行账户信息
- 身份证号

> 任何邮件/网站要求你输入以上信息，先验证对方身份。亚马逊要这些信息只在 `sellercentral.amazon.com` 后台操作，不会通过邮件链接。

#### 特征 4：发件人邮箱伪装

邮件发件人显示名可以随便写，但**邮箱地址骗不了人**。

```
显示名：Amazon Seller Central        ← 骗你的眼睛
邮箱地址：noreply@amazon-notice.com   ← 骗不了，主域名不是 amazon

正规的发件邮箱：
  ✅ no-reply@sellercentral.amazon.com
  ✅ no-reply@email.amazon.com
  ✅ @amazon.com / @amazon.de 等亚马逊域名
```

#### 特征 5：链接文字和真实地址不一致

邮件里显示「点击登录亚马逊后台」，但你把鼠标悬停在上面（不要点！），浏览器底部会显示真实地址：

```
显示文字：登录亚马逊卖家后台
真实地址：http://amazon.secure-login-verify.top/phishing
                                    ↑ 这是钓鱼网站
```

> **永远不要相信链接文字，只相信鼠标悬停显示的真实地址。**

### 三步验证法（遇到任何可疑链接都这样做）

```
第一步：停（不要点）
  收到要求你"登录""验证""更新信息"的邮件/短信/消息
  → 手指停下，不要点链接

第二步：看（检查 3 样东西）
  ① 发件人邮箱地址（不是显示名）→ 主域名是不是 amazon？
  ② 链接真实地址（鼠标悬停，不点击）→ 主域名是不是 amazon？
  ③ 内容语气 → 有没有紧迫感/恐吓/威胁？

第三步：验（独立验证）
  不点邮件里的链接，自己打开浏览器
  手动输入 sellercentral.amazon.com 登录
  → 如果真有问题，后台会显示通知
  → 后台没通知 = 邮件是假的
```

> **核心原则：永远通过自己手动输入的官方网址进入后台，不走邮件链接。** 这一条就能挡住 99% 的钓鱼攻击。

### 亚马逊运营常见钓鱼场景

| 场景 | 钓鱼话术 | 真相 |
|---|---|---|
| **账号验证** | 「您的卖家账号需要立即验证，否则将被暂停」 | 亚马逊不会邮件链接验证，后台会通知 |
| **付款失败** | 「买家付款失败，请点击确认收款信息」 | 订单付款在后台处理，不需要外部链接 |
| **知识产权投诉** | 「您被投诉侵权，点击查看详情并申诉」 | 侵权通知在后台 Performance → Account Health |
| **退货通知** | 「买家申请退货，点击处理」 | 退货在后台 Manage Returns 处理 |
| **Bestie Review** | 「您有一条新的买家消息待回复」 | 买家消息在后台 Buyer-Seller Messaging |
| **跟卖警告** | 「检测到有人跟卖您的产品，点击查看」 | 第三方工具可能有此功能，但先验证域名 |

### 被钓鱼了怎么办（应急步骤）

如果你已经点了链接甚至输入了信息：

```
1. 立即改密码
   → 用正确的官方网址登录 amazon 后台
   → Settings → Login Settings → 修改密码

2. 开启/重置二次验证（2FA）
   → 如果还没开 2FA，立刻开
   → 如果验证码可能泄露了，重置 2FA

3. 检查后台异常
   → 检查是否有陌生的银行账户/收款方式被添加
   → 检查 Listing 是否被篡改
   → 检查是否有异常订单

4. 联系亚马逊卖家支持
   → 后台 Help → Contact Us
   → 报告疑似钓鱼，让亚马逊协助检查账号安全
```

## 实操要点

- **走官方入口**：需要登录后台时，永远手动输入 `sellercentral.amazon.com`，不走任何邮件链接。
- **悬停看真身**：看到任何链接，先鼠标悬停（不点）看真实地址。
- **看发件人邮箱地址**：不是看显示名，点开发件人详情看真实邮箱。
- **紧迫感 = 危险信号**：「24小时内」「立即」「否则封号」这种话术几乎都是钓鱼。
- **绝不通过邮件链接输入密码/验证码/银行卡**：正规平台不这么做。
- **不确定就联系官方**：通过后台 Help → Contact Us 问亚马逊，不要回邮件。

## 动手练习

拿你最近收到的 3 封「看起来像亚马逊」的邮件，用三步验证法逐一检查。重点看：
1. 发件人真实邮箱地址是什么？
2. 邮件里链接的真实地址（悬停看）是什么？
3. 有没有紧迫感话术？

你会发现至少有 1 封经不起验证。
