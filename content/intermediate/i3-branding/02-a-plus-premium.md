# 中级3.2 A+ 高级版与品牌故事

> **中级3：品牌化基础**
> **本课目标**：学完能判断自己是否符合 Premium A+ 申请条件，设计高转化的 A+ 模块，并用品牌故事模块建立品牌叙事，把 Listing 转化率提升 10-20%。

## 这节课解决什么问题

初级课教过基础 A+ 怎么做。但中级要解决的是：基础 A+ 和 Premium A+ 有什么区别？什么产品该用 Premium？品牌故事模块怎么设计？

很多运营的真实状态是：做了基础 A+ 就觉得"图文版有了"，从来不升级 Premium A+，白白浪费了一个能提升转化率 10-20% 的武器。或者用了 Premium A+ 但模块设计混乱，反而拉低转化。

这节课讲清 Premium A+ 的申请条件、模块设计原则、品牌叙事逻辑，让你的 A+ 从"有"升级到"好"，真正服务于转化。

## 核心知识

### 基础 A+ vs Premium A+ 的区别

| 维度 | 基础 A+（标准版） | Premium A+（高级版） |
|---|---|---|
| 可用模块数 | 最多 5 个 | 最多 15 个 |
| 模块类型 | 标准图文模块 | 含视频、轮播、对比表、交互模块 |
| 品牌故事 | 不含 | **含品牌故事模块** |
| 移动端体验 | 一般 | **优化移动端** |
| 转化率提升 | 5-10% | **10-20%** |
| 费用 | 免费 | 免费（但有门槛） |

> 核心认知：**Premium A+ 不是"更好看的 A+"，是"能讲更完整故事、做更复杂转化的 A+"**。它多了视频、轮播、对比表、品牌故事，能把 Listing 从"产品页"升级到"品牌体验页"。

### Premium A+ 的申请条件

Premium A+ 不是想用就能用，必须满足亚马逊的邀请条件：

| 条件 | 具体标准 |
|---|---|
| **① 品牌备案** | 已完成 Brand Registry |
| **② A+ 使用历史** | 过去 6 个月持续使用基础 A+ |
| **③ Catalog 规模** | 品牌下有一定数量的 Listing（通常 ≥ 10 个 ASIN 启用 A+） |
| **④ 邀请制** | 亚马逊会主动发邀请到品牌备案邮箱 |

```
如何知道自己是否符合：
  → 后台 → Advertising → A+ Content
  → 看是否有「Premium A+」选项 / 「Enhanced Brand Features」
  → 如果灰显 → 还没达标，继续用基础A+
  → 如果可选 → 已达标，可以申请

未达标的补救：
  ① 给品牌下所有Listing都启用基础A+（扩大Catalog）
  ② 持续运营6个月以上
  ③ 等亚马逊邀请邮件
  ⚠️ 没有捷径，只能积累
```

### 基础 A+ 的模块设计原则

即使用基础 A+，模块设计也要有逻辑，不是"有什么模块填什么"：

| 模块顺序 | 内容 | 目的 |
|---|---|---|
| **模块1：品牌横幅** | 品牌 logo + 一句品牌口号 | 建立品牌认知 |
| **模块2：核心卖点图** | 3-4 个核心功能 + 场景图 | 强化产品价值 |
| **模块3：对比表** | 跟竞品的功能对比 | 建立差异化优势 |
| **模块4：使用场景** | 实际使用场景图 | 激发购买想象 |
| **模块5：品牌信任** | 资质认证/质检/保修 | 打消购买顾虑 |

```
基础A+设计的「三不要」：
✗ 不要用纯文字模块（没图，没人看）
✗ 不要堆砌功能参数（客户看不懂）
✗ 不要用跟主图重复的图（浪费位置）

基础A+设计的「三要」：
✓ 每个模块都要有图（视觉优先）
✓ 卖点要翻译成「客户利益」（不是「3000mAh」而是「充电1次用3天」）
✓ 对比表要突出你的优势项（别暴露劣势）
```

### Premium A+ 的高价值模块

Premium A+ 解锁的模块里，有几个是转化利器：

| 模块类型 | 作用 | 适用场景 |
|---|---|---|
| **视频模块** | 产品展示/使用教程/开箱 | 需要展示动态效果的产品 |
| **轮播图模块** | 多张图滑动展示 | 多颜色/多款式/多功能 |
| **对比表模块** | 跟自家其他产品对比 | 产品线内引导 |
| **交互模块** | 点击展开/问答 | 复杂产品的信息分层 |
| **品牌故事模块** | 品牌叙事 | 建立品牌认同（见下节） |

> 转化率最高的两个模块：**视频模块**和**对比表模块**。视频能展示产品真实效果，打消"货不对板"顾虑；对比表能引导客户买更贵的产品（向上销售）。

### 品牌故事模块的设计逻辑

品牌故事是 Premium A+ 独有的模块，出现在 Listing 中部，是建立品牌认同的核心位置：

| 品牌故事结构 | 内容 | 目的 |
|---|---|---|
| **第1屏：品牌使命** | 我们为什么做这个品牌 | 建立情感连接 |
| **第2屏：产品线展示** | 品牌下的产品矩阵 | 展示品牌实力 |
| **第3屏：差异化** | 我们跟别人有什么不同 | 建立独特认知 |
| **第4屏：信任背书** | 资质/认证/客户评价 | 打消顾虑 |

```
品牌故事的「黄金公式」：

[痛点共鸣] + [我们的解决方案] + [为什么信我们] + [行动召唤]

示例（户外水杯品牌）：
第1屏：「每一次户外，都值得一口好水」——痛点共鸣
第2屏：展示全系列水杯（便携/保温/过滤）——产品实力
第3屏：「军用级材质，10年质保」——差异化
第4屏：FDA认证 + 10万+户外爱好者的选择——信任背书

⚠️ 品牌故事的雷区：
✗ 不要写「我们公司成立于XX年」（客户不在乎）
✗ 不要堆砌公司荣誉（跟客户无关）
✗ 不要用过于官方的话术（像企业官网）
✓ 要讲「客户能得到什么」（利益导向）
```

### A+ 模块的转化优化技巧

A+ 做完不是结束，要持续优化转化：

| 优化项 | 方法 | 效果 |
|---|---|---|
| **卖点翻译成利益** | 「3000mAh 电池」→「充一次用 3 天」 | 客户更容易理解 |
| **对比表突出优势** | 只放你强的对比项，劣势项不放 | 建立差异化 |
| **场景图真实化** | 用真实使用场景，不用纯白底图 | 激发购买想象 |
| **视频前 3 秒抓人** | 视频开头直接展示核心卖点 | 降低跳出 |
| **移动端测试** | 手机端查看 A+ 是否清晰 | 移动端占流量 50%+ |

> 关键数据：**移动端流量占比已经超过 50%**。A+ 设计必须用手机预览，确保文字够大、图片清晰、模块间距合理。很多 A+ 在电脑端好看，手机端一团糟。

### A+ 与 Listing 其他要素的配合

A+ 不是孤立的，要跟 Listing 其他要素配合：

```
Listing 转化要素的分工：

主图：吸引点击（CTR）
标题：匹配搜索词（流量）
五点：传递核心卖点（信息）
A+：建立信任和品牌（转化）
评论：社会证明（转化）

→ A+ 的核心角色是「建立信任和品牌」
→ 主图和标题解决「点不点、来不来」
→ A+ 解决「来了买不买」
→ 所以A+要重点展示：场景、对比、信任背书、品牌故事
→ 这些是主图和五点装不下的内容
```

## 实操要点

- **优先升级 Premium A+**：转化率比基础 A+ 高 10-20%，符合条件就申请。
- **A+ 模块要有逻辑顺序**：品牌横幅→核心卖点→对比表→使用场景→品牌信任。
- **卖点翻译成客户利益**：不写参数写结果，不写「3000mAh」写「充一次用3天」。
- **品牌故事用黄金公式**：痛点共鸣+解决方案+为什么信我们+行动召唤。
- **必须测移动端**：手机端流量占 50%+，A+ 在手机上要清晰好看。

## 动手练习

选你一个有 A+ 的产品，做一次「A+ 体检」：①列出当前用的所有模块及顺序，对照黄金顺序看是否合理；②检查每个卖点是否翻译成了客户利益（把参数改成结果）；③用手机预览 A+，看文字是否够大、图片是否清晰；④如果你符合 Premium A+ 条件，规划品牌故事模块的四屏内容。目标是让你的 A+ 从"有内容"升级到"会转化"。
