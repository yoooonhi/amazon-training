# 初级3.4 补货计划（实操）

> **初级3：库存精细化运营**
> **本课目标**：能把前三课的公式和判断串起来，用一个标准补货计划表模板，对一个 SKU 输出一份完整的、可直接执行的补货计划。

## 这节课解决什么问题

前三课你学了三样东西：怎么算补货量（3.1）、怎么算资金占用和分配（3.2）、怎么识别和处理滞销（3.3）。但学的时候是分开的，真到干活，这三件事是拧在一起的——你不可能先算完补货量、再去想钱够不够、最后才想起来看看这 SKU 是不是其实已经滞销了。

实际工作中，运营每周要面对的是一堆 SKU，每个都要回答一连串问题：这个现在多少货？还能卖几天？要不要补？补多少？钱够吗？什么时候下单能赶上？什么时候该清而不是补？如果这些问题靠脑子一个一个想，要么漏、要么乱，最后一周过去补货计划还是空的。

这节课给你一张**补货计划表**模板，把日均销量、当前库存、在途、可售天数、建议补货量、资金占用、预计到货日全部装进去。你只要按列填数、按规则判断，每个 SKU 都能输出一份完整计划。学完你能每周用这张表把整盘货过一遍，补哪些、清哪些、钱怎么分，一目了然。

## 核心知识

### 补货计划表模板（核心交付物）

这张表就是这节课的全部。一张表搞定所有判断。建议在 Excel / 飞书表格里搭出来，每周更新。

| 列 | 字段 | 数据来源/算法 |
|---|---|---|
| A | SKU | 后台 |
| B | 近30天日均销量 | 后台销量÷30 |
| C | 近7天日均销量 | 后台销量÷7（交叉验证用） |
| D | 预估日均销量 | 旺季按去年×增长；淡季用30天日均带趋势（3.1） |
| E | 当前可售库存 | 后台（只算可售） |
| F | 在途库存 | 后台（逐批核状态，3.1） |
| G | 可售天数 | E ÷ D（3.3预警用） |
| H | 库销比 | E ÷(B×30)（3.3滞销预警） |
| I | 补货周期天数 | 完整链条（3.1：生产+海运+清关+上架） |
| J | 安全库存天数 | 按风险等级（3.1） |
| K | 建议补货量 | D×I + D×J − E − F（3.1公式） |
| L | 单件成本 | 采购价 |
| M | 本批资金占用 | K × L（3.2） |
| N | 预计下单日 | 今天 |
| O | 预计到货上架日 | 今天 + I |
| P | 处理动作 | 补货/暂缓/清仓/弃置（综合G、H判断） |

### 用一张表走一遍完整案例

假设今天是 10 月 1 日，黑五在 11 月底（约 60 天后）。我们对一个主力 SKU 做完整计划。

```
SKU-A 完整补货计划（10月1日制定）

B. 近30天日均：18 件
C. 近7天日均：22 件（在涨，旺季前流量起来了）
D. 预估日均：去年旺季日均50 × 今年增长1.2 × 旺季加成1.2 = 72 件
   （旺季前60天，必须用旺季预估，不能用18或22）
E. 当前可售库存：500 件
F. 在途库存：1500 件（已核状态，确实在途）
G. 可售天数（按旺季预估D）：(500+1500) ÷ 72 ≈ 28 天
   （注意：用旺季日均算，只有28天，很紧张！）
H. 库销比：500 ÷ (18×30) = 0.9（淡季看库销比很健康，但旺季要爆单，这数会瞬间恶化）
I. 补货周期：76 天（完整海运链条）
J. 安全库存天数：20 天（旺季爆款高风险）
K. 建议补货量 = 72×76 + 72×20 − 500 − 1500
              = 5472 + 1440 − 500 − 1500
              = 4912 件（凑整到整箱，约5000件）
L. 单件成本：30 元
M. 本批资金占用 = 5000 × 30 = 150,000 元
N. 预计下单日：10月1日（今天，必须立刻下，不能再拖）
O. 预计到货上架日：10月1日 + 76天 = 12月16日
P. 处理动作：立即补货5000件（旺季核心款，断货代价极大）

关键判断：
- 虽然淡季库销比0.9看着健康，但用旺季预估日均算可售天数只有28天，
  而这批货要76天才到——必须现在就下，否则到货前必断货。
- 今天下单12月16日才到，黑五(11月底)其实赶不上完整旺季，
  说明这批已经晚了，旺季高峰可能要靠空运补一批应急。
```

这一步的教训：**旺季备货必须提前 90 天以上启动**。10月1日才为11月底的黑五备货，海运已经来不及了。如果9月初就下，10月中旬能到，正好接旺季。

### 判断逻辑：补货 / 暂缓 / 清仓 / 弃置

表里 P 列"处理动作"怎么定？综合 G（可售天数）和 H（库销比）判断：

```
判断决策树：

1. 先看 H 库销比：
   - 库销比 > 6 且销量下滑 → 走滞销处理流程（3.3），不补货
   - 库销比 4-6 → 启动清仓/促销，暂缓补货
   - 库销比 < 4 → 进入补货判断 ↓

2. 再看 G 可售天数（用旺季/淡季对应预估日均算）：
   - 可售天数 > 补货周期+安全库存 → 暂缓，不急
   - 可售天数 < 补货周期 → 紧急！立即下单，可能还要空运救场
   - 中间 → 正常补货，按公式量下单

3. 最后看资金（M列）：
   - 本批资金占用超预算 → 按3.2排序法砍边缘SKU，保核心
   - 资金够 → 按计划下单
```

### 把多个 SKU 汇总成资金预算

单个 SKU 的表算完后，把所有 SKU 的 M 列（本批资金占用）加总，和你的总预算对比：

```
本周补货汇总（10月1日）

SKU    建议补货量  资金占用   动作
A      5000       150,000   立即补（旺季核心）
B      2000        40,000   正常补
C      1500        30,000   正常补
D      800         16,000   少补（周转95天，勤补）
E      0                0   不补（库销比5，先Coupon）
F      0                0   不补（周转200天，弃置评估）
─────────────────────────────────
合计              236,000

总预算：250,000 元
→ 够，按计划执行，剩余14,000留作应急
```

如果合计超预算，就回到 3.2 的排序法：按周转×利润×战略砍边缘 SKU，保核心爆款不断货。

### 一份计划的生命周期

补货计划不是做完就完，要跟踪到货到上架：

```
Day 0   制定计划，下单给工厂
Day 7   跟踪工厂排产进度
Day 30  跟踪生产/质检完成
Day 40  确认发货、海运船期
Day 70  跟踪到港清关
Day 76  FBA接收上架 → 计划闭环

期间任何环节延误 → 更新O列预计到货日，判断会不会断货，
  会断 → 启动空运应急补货
```

## 实操要点

- **每周固定一天更新整张表**（比如周一），不要每天零散改，效率低还容易乱。
- **旺季用旺季预估日均、淡季用淡季日均**，两个季节的可售天数判断完全不同，别混。
- **每个 SKU 走决策树**：先库销比（要不要清）、再可售天数（要不要补）、再资金（补不补得起），三步不能乱。
- **预计到货日要持续更新**：海运延误是常态，到货日一变，断货风险就变，要动态调整。
- **旺季备货提前90天启动**：海运76天+缓冲，黑五/圣诞的货最迟9月就要下单，10月再下就晚了。

## 动手练习

选你手上 3-5 个真实 SKU，用补货计划表模板做一份本周补货计划：

1. 在表格里填满 A-O 列（日均、库存、在途、可售天数、库销比、补货量、资金占用、到货日）。
2. 对每个 SKU 走决策树，定 P 列动作（补货/暂缓/清仓/弃置）。
3. 把所有 SKU 的资金占用加总，和你的总预算对比，超了就用排序法砍。
4. 检查最紧急的 SKU：今天下单，到货日是哪天？会不会断货？要不要搭配空运？
5. 把这份计划保存下来，下周同一天再更新一遍，对比实际进展。

做完你就拥有了一个能每周复用的库存管理工具——这才是"精细化运营"落地的东西。
