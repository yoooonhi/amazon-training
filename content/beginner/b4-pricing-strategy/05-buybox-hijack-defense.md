# 初级4.5 Buy Box 防御与跟卖应对

> **初级4：定价与促销策略**
> **本课目标**：理解 Buy Box 不是"你的"而是"你每天要保住的"，会排查 Buy Box 丢失的原因，掌握跟卖的标准应对流程。

## 这节课解决什么问题

你可能遇到过这种事：广告还在跑，预算没动，排名也没掉，但**销量突然腰斩**——查半天才发现，详情页的"Add to Cart"按钮旁边的卖家不是你了。Buy Box（黄金购物车）丢了。

Buy Box 是亚马逊详情页上那个"加入购物车"按钮的归属权。**大约 80% 的销量走 Buy Box**，谁拥有它，谁就拿到这 80% 的流量转化。你丢了 Buy Box，哪怕排名还在，买家点进来加进购物车的也是别人的产品。

Buy Box 丢失最常见、也最让人头疼的原因是**被跟卖**——别的卖家在你的 ASIN 下挂了个 offer，用比你低的价格抢走了购物车。这节课把 Buy Box 的运作机制、丢失原因、跟卖应对讲清楚。

## 核心知识

### Buy Box 的归属逻辑

Buy Box 不是固定属于某个卖家的，亚马逊用一个算法实时决定"现在谁的产品出现在购物车按钮里"。影响算法的因素：

```
Buy Box 归属的 5 大因素（权重从高到低）：

① 价格（到手价，不是标价）
   - 含运费的总价最低者优先
   - 注意是"到手价"——同样标价，FBA 比 FBM 到手价低（免运费）

② 配送方式
   - FBA >> FBM（FBA 几乎总是赢 FBM，即使价格略高）
   - Prime 标记是巨大的 Buy Box 加成

③ 账号健康
   - 订单缺陷率（ODR）、迟发率、取消率
   - 健康度差的卖家会被剥夺 Buy Box 资格

④ 销量与转化历史
   - 卖得多、转化好的卖家有优势
   - 这就是为什么新品不容易拿 Buy Box

⑤ 发货速度/库存
   - 有库存、发货快的优先
   - 断货会直接丢 Buy Box
```

> 关键认知：**Buy Box 是"挣来的"，不是"固有的"。** 你的 ASIN，不代表购物车永远是你的——任何一个因素恶化，Buy Box 就可能被别人抢走。

### Buy Box 丢失的 5 种原因排查法

发现销量下滑，第一反应应该是检查 Buy Box。排查顺序：

| 原因 | 怎么查 | 怎么解决 |
|---|---|---|
| **① 被跟卖** | 详情页看 "Other Sellers" 有没有别人 | 见下文跟卖应对阶梯 |
| **② 价格被竞品压低** | 你和跟卖者/竞品的到手价对比 | 降价或匹配（但要守住成本线） |
| **③ 库存将断货** | 后台看可售库存 + 预计断货日 | 紧急补货 / 暂时提价减速 |
| **④ 配送降级** | 是否从 FBA 变成 FBM？库存转 FBM？ | 恢复 FBA / 检查货件状态 |
| **⑤ 账号绩效恶化** | Account Health 看 ODR/迟发率 | 修复绩效（处理退货、申诉） |

```
Buy Box 丢失排查清单（按这个顺序走）：
□ 详情页有没有 Other Sellers？（被跟卖）
□ 我的价格 vs 其他卖家的到手价？（价格被压）
□ 库存还够吗？预计什么时候断货？（库存）
□ 配送方式有没有变？FBA 库存还在吗？（配送）
□ Account Health 有没有红旗？（绩效）
```

> 最常被忽略：**断货前夕会丢 Buy Box。** 亚马逊知道你快没货了，会把购物车转给有库存的卖家。所以补货（见初级3.1）不只是防断货，也是防丢 Buy Box。

### 跟卖：最棘手的 Buy Box 威胁

跟卖（Hijack）指别的卖家在你的 ASIN 下创建 offer，用更低价格抢购物车。跟卖分两种性质：

| 类型 | 特征 | 应对难度 |
|---|---|---|
| **正品跟卖** | 卖的是和你一样的正品（可能是你的分销商/经销商失控） | 中——沟通+控价 |
| **假货跟卖** | 卖的是仿品，蹭你的 Listing 流量 | 高——必须用品牌工具打击 |

> 警告：**跟卖最大的危害不只是抢购物车，而是假货跟卖会毁掉你的评论和品牌。** 假货质量差，买家收到后留差评——差评挂在你的 Listing 上（因为是同一个 ASIN），你的转化和星级一起崩。

### 跟卖应对阶梯（从轻到重）

发现跟卖，按这个阶梯逐步升级，不要一上来就动用最重的手段：

```
跟卖应对五阶梯：

① 监控（日常预防）
   - 每天检查核心 ASIN 有没有 Other Sellers
   - 用工具（如 Keepa 的卖家追踪）或后台 Brand Analytics
   - 早发现是关键，跟卖者待得越久越难赶

② 警告信（试探性）
   - 通过买家消息给跟卖者发警告
   - 声明你是品牌方/版权方，要求停止销售
   - 对胆小的跟卖者有效，对老油条无效

③ Test Buy（取证）
   - 自己下单买跟卖者的产品
   - 收到后拍照取证（证明是假货/侵权）
   - 这是后续投诉的关键证据

④ 品牌备案投诉（正式打击）
   - 用 Test Buy 的证据，通过品牌备案后台提交侵权投诉
   - 版权/商标侵权投诉，亚马逊会下架跟卖者的 offer
   - 这是打击假货跟卖最有效的手段

⑤ Transparency / Project Zero（长效防御）
   - Transparency：给产品贴唯一溯源码，没有码的跟卖无法上架
   - Project Zero：自助删除疑似假货 listing
   - 这是从源头杜绝跟卖的根本手段（见高级课）
```

| 跟卖场景 | 推荐应对 |
|---|---|
| 正品分销商失控跟卖 | 沟通控价，必要时断供 |
| 偶尔出现的小跟卖 | 警告信往往就吓退 |
| 顽固的假货跟卖 | Test Buy 取证 → 品牌备案投诉 |
| 反复出现的职业跟卖 | 上 Transparency 一劳永逸 |

> 认知：**赶跟卖是"防御战"不是"一次性的"。** 今天赶走一个，明天可能又来一个。长期做品牌的产品，最终都要上 Transparency 这类源头防御工具，否则永远在被动应付。

### 日常 Buy Box 防护意识

与其丢了 Buy Box 再救，不如日常就守住：

```
Buy Box 日常防护三件事：

① 每天看一眼核心 ASIN 的购物车
   - 加进每日运营流程（见入门6 的日常清单）
   - 30 秒就能确认，但能避免大损失

② 价格保持竞争力
   - 定期查竞品到手价（见初级4.1）
   - 不要比市场高太多，否则即使没跟卖也会丢购物车

③ 库存永远不断
   - 补货模型（初级3.1）不只是防断货，也是防丢购物车
   - 安全库存要算足，旺季加码
```

> 铁律：**Buy Box 丢失的黄金救援时间是 24 小时内。** 超过一天，你的销量、排名、广告表现都会被拖累。所以"每天看一眼"不是多余，是止损。

## 实操要点

- **Buy Box 是挣来的不是固有的**：价格、配送、库存、绩效、销量共同决定归属，任何一项恶化都可能丢购物车。
- **销量异常先查 Buy Box**：广告没动、排名没掉却销量腰斩，第一反应是看购物车还在不在。
- **丢购物车按 5 因素排查**：跟卖→价格→库存→配送→绩效，按顺序走。
- **跟卖应对要分级**：监控→警告→Test Buy→品牌投诉→Transparency，从轻到重，假货跟卖必须取证后投诉。
- **日常防护胜过急救**：每天看一眼购物车、价格保持竞争力、库存不断，比丢了再救省十倍力气。

## 动手练习

挑你的一个主力 ASIN，做一次"Buy Box 健康检查"：

1. 打开这个 ASIN 的详情页，确认"Add to Cart"按钮旁的卖家是不是你。往下看有没有 "Other Sellers" 区域——如果有别人，记录他们的价格和配送方式（FBA 还是 FBM）。
2. 对照本课的 5 因素，自检你这个 ASIN 的 Buy Box 风险：价格有竞争力吗？库存还够撑多久？配送是 FBA 吗？账号 Account Health 有没有红旗？
3. 如果发现有跟卖——判断是正品还是假货，规划你的应对阶梯：先发警告信还是直接 Test Buy 取证？你有没有品牌备案可以投诉？
4. 把"每天检查 Buy Box"加进你的日常运营清单（参考初级1.5 的周度清单），养成习惯。
