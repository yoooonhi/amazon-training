# 初级2.3 Search Terms 持续迭代

> **初级2：Listing持续优化**
> **本课目标**：能从搜索词报表反向找出新词补进 backend keywords，并知道什么时候该小改、什么时候该大改、改完怎么验证效果。

## 这节课解决什么问题

入门课你学过 Search Terms 怎么写：250 字节、空格分隔、不重复标题词。上线那一刻你写得挺好。但问题是——**Search Terms 不是设一次就不管了**。

三个月后情况变了：买家开始用你没想过的新词搜你的产品（比如某个新场景词突然火了），但你后台 Search Terms 里没有这个词，A9 匹配不到，白白损失流量。同时，你当初塞进去的一些词，三个月过去发现根本没带来任何曝光，纯粹占着 250 字节的宝贵位置。

这节课就解决两件事：**怎么从数据里发现该补/该删的词**，和**怎么改才不会越改越糟**。Search Terms 是个"活"的字段，每个季度至少要迭代一次。

## 核心知识

### 先复习：Search Terms 的硬约束

入门课讲过，这里再强调一次，因为迭代时最容易踩这些坑：

| 约束 | 规则 | 踩坑后果 |
|------|------|---------|
| 字节上限 | **250 字节**（UTF-8），建议留 5-10 字节余量 | 超限**整段作废**，不是截断 |
| 分隔符 | 空格，**不用逗号** | 逗号割裂词组还浪费字节 |
| 大小写 | 全小写 | 大写不加分还乱 |
| 重复 | 不重复标题/五点已有的词 | 重复不双重加权，纯浪费字节 |
| 禁忌 | 不写竞品品牌、不写促销词、不写主观词 | 侵权/违规风险 |

> 迭代时最容易踩的坑：**加新词忘了删旧词，导致超 250 字节**。每次加词必须先算总字节，超了就先删再补。

### 迭代的核心逻辑：从搜索词报表反推

Search Terms 迭代的唯一数据源是**搜索词报表**（Search Term Report，来自广告后台）。逻辑是：

```
报表里出现的搜索词 = 买家真实在搜的词
                  ↓
如果这个词出单好，但你的 Search Terms 里没有 → 补进去
如果这个词在你的 Search Terms 里，但 30 天 0 曝光 → 删掉换新的
```

### 第一步：找该补的词（新增）

打开最近 **30 天**搜索词报表，按这个标准筛"该补进 Search Terms 的词"：

| 筛选条件 | 阈值 | 为什么 |
|---------|------|-------|
| 转化好 | 出单 ≥ 3 单 或 转化率 ≥ 5% | 证明这个词有效 |
| 搜索量适中 | 月搜 1000-10000 | 太大竞争激烈，太小没意义 |
| **当前 Search Terms 没有** | 反查后台字段确认 | 已有的不用补 |
| 标题/五点没有 | 反查前台文案确认 | 前台已有的不用补（重复无效）|

> 关键：只补"前台没有"的词。前台已经出现的词，再塞进 Search Terms 不会双重加权，纯浪费字节。

### 第二步：找该删的词（清理）

同样看搜索词报表，但反过来——找"占着位置不出活"的词：

| 情况 | 判断标准 | 动作 |
|------|---------|------|
| 完全没曝光 | Search Terms 里的词，在报表里 30 天 0 曝光 | 删，换新词 |
| 有曝光没转化 | 曝光 >1000 但 0 出单 | 删（不相关或转化差）|
| 相关性变低 | 产品方向调整，某类词不再相关 | 删 |

### 第三步：算字节，先删后加

这一步是新手最容易翻车的。**永远先删再加**，删完腾出字节再补新词：

```
当前 Search Terms：235 字节
想加的新词：running podcast（17 字节）
→ 直接加 = 252 字节，超限整段作废！

正确流程：
1. 先删 2 个不出单的旧词（腾出 25 字节）
2. 现在剩 210 字节
3. 再加 running podcast（17 字节）
4. 新总字节：227，留 23 字节余量 ✓
```

### 字节计算工具

| 工具 | 准确度 | 用法 |
|------|--------|------|
| 亚马逊后台提交时显示 | **最准**（官方） | 改完保存时看提示 |
| Helium 10 Index Checker | 准 | 批量检查是否被索引 |
| 在线 UTF-8 字节计数器 | 准 | 搜 "UTF-8 byte counter" |
| 脑数 | **不准** | 千万别用 |

> 非英文字符（西语 ñ、法语 é、中文）占 2-4 字节。你以为 200 字符可能实际 300+ 字节。**永远用工具数**。

### 什么时候大改 vs 小改

这是迭代的节奏判断：

| 改动类型 | 触发条件 | 频率 | 风险 |
|---------|---------|------|------|
| **小改（微调）** | 换 2-5 个词，字节变动 < 30 | 每月 1 次 | 低 |
| **大改（重构）** | 换掉一半以上词，或方向调整 | 每季度 1 次 | 中（可能短期排名波动）|

> 纪律：**大改之间至少间隔 4 周**。改完要给 A9 算法 2-4 周时间重新学习和匹配，频繁大改会让排名剧烈波动，越改越乱。

### 改完怎么验证：三看

改完 Search Terms 后，**至少等 2 周**再判断效果，看三个指标：

| 指标 | 在哪看 | 健康信号 |
|------|--------|---------|
| **总曝光量** | 业务报告 / 广告报表 | 比改之前涨了（说明新词被索引到了）|
| **新增词的曝光** | 搜索词报表 | 新补的词开始出现曝光 |
| **总销量** | 业务报告 | 不掉（掉说明改坏了，回滚）|

如果改完 2 周总曝光和销量都掉，**立即回滚到改之前的版本**——说明你删错了词或加错了词。

### 什么时候该做 Listing Index 检查

每季度做一次**全词索引检查**：用 Helium 10 的 Index Checker 或免费的卖家工具，检查你的核心词池里有多少词**真的被亚马逊索引到了**（能被搜到）。

```
理想状态：核心词池 80% 以上被索引
警告状态：核心词池 < 50% 被索引
        → 说明标题/Search Terms 没写好，大量词匹配不到
        → 需要重构 Search Terms
```

## 实操要点

- **Search Terms 是活的，每季度至少迭代一次**，不是设一次就不管。
- **唯一数据源是搜索词报表**：补词找转化好的新词，删词找 30 天 0 曝光的死词。
- **只补前台没有的词**：前台已有的重复塞 Search Terms 不加权，纯浪费字节。
- **永远先删后加**：加词前先算总字节，超 250 整段作废。
- **小改每月一次，大改每季度一次**，两次大改间隔至少 4 周给算法学习时间。
- **改完等 2 周再判断**，看曝光和销量，掉了立即回滚。
- **每季度做一次 Index 检查**：核心词索引率应 >80%，低于 50% 必须重构。

## 动手练习

拿你一个上线超过 3 个月的产品，做一次完整的 Search Terms 迭代：

1. **导数据**：下载最近 30 天搜索词报表，按出单数降序。
2. **找补词**：挑出 5 个"出单 ≥3、转化好、当前 Search Terms 和前台都没有"的词，记下来。
3. **找删词**：把当前 Search Terms 里的词逐个去报表里搜，找出 30 天 0 曝光的死词，记下来。
4. **算字节**：用在线 UTF-8 字节计数器，先删死词算出新字节，再加新词，确认总数 ≤ 245 字节。
5. **改后台**：保存新版本，记下改动日期。
6. **2 周后验证**：对比改前后的总曝光和销量，掉了就回滚。

如果暂时没产品可练，找一个产品的 Search Terms 字段，用字节计数器数一遍当前字节，找出有没有超 250、有没有逗号、有没有重复标题词——光是这一步就能发现很多问题。
