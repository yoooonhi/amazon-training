# 初级1.4 ACOS 压降闭环

> **初级1：广告优化实战**
> **本课目标**：掌握一个完整的"降 ACOS"工作流——会先诊断根因（CTR / CVR / CPC 哪个的问题），再对症下药，最后验证——而不是只会"降价"这一个动作。

## 这节课解决什么问题

"ACOS 太高了怎么办？"——这是运营被问得最多的问题。入门课的标准答案是"降价或否定"。但你会发现：很多时候你降价了 ACOS 反而更高，因为降价后广告跑不到好位置，转化更差，陷入死循环。

问题出在哪？**ACOS 不是一个独立的毛病，它是三个变量的结果**。ACOS = (CPC ÷ CVR) ÷ 客单价，你只会动其中一个（CPC，通过降价），当然解决不了另外两个（CVR 和点击量）引起的 ACOS 高。

这节课给你一个**完整的 ACOS 压降闭环**：先定位是哪个变量拖累了 ACOS，再针对那个变量对症下药，最后用数据验证有没有压下来。**核心是诊断，不是降价。** 盲目降价是最差的选择。

## 核心知识

### 先记住 ACOS 的拆解公式

这是整节课的地基：

```
ACOS = 广告花费 ÷ 广告销售额
     = (CPC × 点击数) ÷ (点击数 × CVR × 客单价)
     = CPC ÷ (CVR × 客单价)

→ ACOS 高，只可能是这三个里至少一个出问题：
  ① CPC 太高
  ② CVR 太低
  ③ 客单价太低（这个一般不靠广告解决，靠选品和定价）
```

> 降价只能动 ①CPC。如果根因是 ②CVR，你降再多 CPC 也压不下来——而且会越降越糟，因为跑不到好位置，CVR 只会更低。

### ACOS 高的根因诊断树

遇到 ACOS 高的广告活动，**先别动手，先诊断**。按这棵树走一遍：

```
ACOS 高
  │
  ├─ 第一步：CTR 正常吗？（参考：站点均值 0.2%-0.5%，标品偏低）
  │    │
  │    ├─ CTR 偏低（< 0.2%）→ ②问题：主图/标题/价格不吸引人
  │    │     → 根因不在广告，在 Listing。改主图、优化标题、查价格。
  │    │
  │    └─ CTR 正常 → 进入第二步
  │
  ├─ 第二步：CVR 正常吗？（参考：整体 CVR 行业均值的 70% 以上）
  │    │
  │    ├─ CVR 偏低 → ③问题：Listing 内功 / 价格 / 评价 / 流量精准度
  │    │     → 查：A+页面、评价星级和数量、价格vs竞品、搜索词相关性
  │    │
  │    └─ CVR 正常 → 进入第三步
  │
  └─ 第三步：CPC 偏高吗？
       │
       └─ CPC 高 → ①问题：匹配类型太宽 / 出价虚高 / 词太热
            → 动作：收匹配类型（宽→词组→精准）、降出价、砍热词
```

### 三种问题的诊断对照表

| 根因 | 报表特征 | 典型数字 | 对症动作 |
|---|---|---|---|
| **CTR 问题** | 点击率低，但点了的人转化不差 | CTR < 0.2%，CVR 正常 | 改主图、优化标题、降价/促销提吸引度 |
| **CVR 问题** | 点击率正常，但点进来不买 | CTR 正常，CVR < 行业 70% | 查 A+、评价、价格、相关性；收搜索词 |
| **CPC 问题** | CTR 和 CVR 都正常，就是点击太贵 | CTR 正常、CVR 正常、CPC 高于行业 30% | 收匹配类型、降出价、挪到精准活动 |

> 关键纪律：**CTR 和 CVR 都没正常之前，不要去动 CPC。** Listing 内功没修好，你把 CPC 压得再低，单次点击便宜了，但不出单，ACOS 还是无限大（除以 0）。

### 对症下药的具体动作

#### ② CVR 低：查这 5 件事（按优先级）

CVR 低是 ACOS 高最常见的根因，因为 Listing 内功问题最难发现。按这个顺序排查：

| 优先级 | 检查项 | 怎么查 | 动作 |
|---|---|---|---|
| 1 | **评价星级** | 星级 < 4.0？评价 < 15 条？ | CVR 杀手 No.1，优先攒好评 |
| 2 | **价格 vs 竞品** | 搜索核心词看首页，你价格在前 5 名里排第几 | 比首页均价高 15% 以上就会拖 CVR |
| 3 | **A+ / 详情页** | 主图能看懂吗？五点说清卖点吗？ | 自己用手机点开看一遍，5 秒内能不能 get 卖点 |
| 4 | **搜索词相关性** | 跑进来的词跟产品强相关吗？ | 不相关词否定（见初级1.2），相关性差就收匹配类型 |
| 5 | **库存 / 配送** | 是 FBM？或者库存将断货？ | FBM 转化比 FBA 低 30%-50% |

#### ① CPC 高：收匹配类型是首选

很多人 CPC 高第一反应是"降价"，但**收匹配类型往往比降价更有效**：

| 当前匹配 | 问题 | 收紧到 | 效果 |
|---|---|---|---|
| Broad（广泛） | 跑进来一堆不相关长尾，CPC 便宜但不出单 | Phrase（词组） | 流量更精准，CVR 提升 |
| Phrase（词组） | 仍有些漂移词 | Exact（精准） | CPC 可能上升但 CVR 大涨，ACOS 下降 |

> 收匹配类型的逻辑：**精准活动 CPC 通常更高，但因为 CVR 大幅提升，ACOS 反而下降。** 不要因为"精准 CPC 贵"就排斥它——ACOS 才是终点指标。

#### ① CPC 高：出价调整的分档

确认要降 CPC 后，**不要一刀切降价**，分档处理：

```
出价调整纪律：
- 单次降幅 ≤ 20%（一次降太多广告直接掉位，CVR 崩盘）
- 调整后观察 7 天
- 优先降"花钱多但不稳定"的词，核心出单词的出价要稳
- 永远保留至少 1-2 个核心词的高出价，保住基本盘流量
```

### 验证：压降闭环的最后一步

调完之后**必须验证**，否则等于没调。验证规则：

```
验证清单（调整后第 8 天）：
□ ACOS 是否下降到目标值？
  → 是 → 这个根因诊断对了，记录方法
  → 否 → 进入复诊

复诊：
□ 是不是同时有多个根因？（CTR 和 CVR 都低）
□ 是不是动了错误的变量？（CVR 问题却去降 CPC）
□ 是不是观察周期太短？（节假日波动）
→ 重新走诊断树
```

| 调整后的表现 | 判断 |
|---|---|
| ACOS 下降 + 出单量维持 | 成功，记录方法 |
| ACOS 下降 + 出单量大幅下滑 | 降过头了，回退一档 |
| ACOS 没变 | 根因没找对，回诊断树 |
| ACOS 反升 | 动错了变量（常见：CVR 问题去降了 CPC），立即回滚 |

> **最危险的失败模式：ACOS 没降下来，就反复降价，越降越崩。** 如果降了两轮 ACOS 还没动，停下来——根因不在 CPC，去查 CVR 和 CTR。

## 实操要点

- **ACOS 高先诊断，不要先动手**：按"CTR → CVR → CPC"三步诊断树走，定位是哪个变量的问题。
- **CTR 和 CVR 没修好之前，不碰 CPC**：Listing 内功是根，根没修好降价只会越降越糟。
- **CVR 低按 5 件事排查**：评价星级 → 价格 → A+ → 搜索词相关性 → 配送方式，按优先级来。
- **收匹配类型比降价更有效**：Broad → Phrase → Exact，精准活动 CPC 贵但 ACOS 低。
- **降 CPC 单次不超 20%，观察 7 天**：核心出单词的出价要稳，保住基本盘。
- **调完必须验证**：ACOS 没降就是根因没找对，回诊断树复诊，不许反复盲降。

## 动手练习

挑一个 ACOS 超过目标值（比如目标 25%，现在跑到 40%）的广告活动，走一遍完整诊断树：(1) 查 CTR——低于 0.2% 就是 Listing 问题；(2) CTR 正常就查 CVR——按 5 件事（评价/价格/A+/相关性/配送）逐项排查，找出最大短板；(3) 如果 CTR 和 CVR 都正常，那才是 CPC 问题，收匹配类型或分档降价。把诊断结论和动作记下来，8 天后验证 ACOS 有没有下降。如果没下降，说明你诊断错了，回来复诊。
