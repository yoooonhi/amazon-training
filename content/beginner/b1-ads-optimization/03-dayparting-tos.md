# 初级1.3 分时调价与 TOS 抢占

> **初级1：广告优化实战**
> **本课目标**：能用 Placement 报表判断你的流量在哪个时段、哪个位置最值钱，并据此制定"高峰加价 / 低谷降价 / TOS 抢位"的可执行规则——而不是凭感觉调出价。

## 这节课解决什么问题

入门课教过你广告位（Placement）的概念，但很多运营调了两年出价，依然是"感觉今天转化差，把出价降一档"。这种拍脑袋式调价，旺季还能糊弄，淡季就把钱烧光。

真正的分时调价要回答三个问题：**你的买家什么时候下单？你的广告费什么时候花得最值？什么时候该去抢 TOS（Top of Search，首页顶部）？** 这三个问题的答案，全在数据里——不是在你的"感觉"里。

还有一个常见误区：很多人听说"TOS 转化好"，就把所有活动的 Placement 加价拉满（+100% 甚至 +200%）。结果 ACOS 爆炸。**TOS 加价是个有门槛的动作**：不是所有活动都值得抢 TOS，加多少有公式，抢完要验证。这节课把这套判断逻辑讲清楚。

> 注意：亚马逊现在有"日预算规则"和"基于位置的竞价（Location-based bidding）"等多重机制，所有调价动作都要在平台规则框架内做，不能违反日预算的下限要求。

## 核心知识

### 第一步：用 Placement 报表判断"TOS 值不值"

在广告后台下载 **Placement 报表**（报告 → 广告活动 → 投放位置），重点看三列数据：

| 位置 | 含义 | 典型表现 |
|---|---|---|
| **Top of Search（TOS）** | 搜索结果首页顶部 | CTR 最高，CPC 最贵 |
| **Rest of Search（ROS）** | 搜索结果其他位置 | CTR 中等，CPC 中等 |
| **Product Pages（PP）** | 商品详情页 | CTR 最低，CPC 最便宜 |

判断 TOS 值不值得抢，看一个核心指标：**TOS 的 ACOS 是不是低于整体 ACOS**。

```
TOS 抢占判断公式：

TOS CVR / 整体 CVR ≥ 1.3
且
TOS CTR / 整体 CTR ≥ 1.5

→ 两个条件同时满足，TOS 值得加价抢占
→ 只满足一个或都不满足，别抢 TOS，白烧钱
```

| 情况 | TOS CVR 倍数 | TOS CTR 倍数 | 动作 |
|---|---|---|---|
| A | 1.5x | 2.0x | **加价抢 TOS**（+25% 起步） |
| B | 1.1x | 1.8x | 谨慎，先小步加（+10%）观察 |
| C | 0.8x | 1.2x | **别抢**，TOS 转化反而差 |
| D | 0.6x | 0.9x | 反向操作：降低默认出价，让系统少投 TOS |

> CTR 倍数比 CVR 倍数更关键。TOS 天然 CTR 高（位置好），但如果 TOS 的转化率没跟着起来，说明你的 Listing 在 TOS 位置"光点不买"——抢了也是烧钱。

### TOS 加多少：分档加价法

决定要抢 TOS 后，**不要一上来就 +100%**。用分档加价：

| TOS 表现 | 起始加价 | 验证周期 | 上限 |
|---|---|---|---|
| 第一次测试 TOS | **+25%** | 7 天 | 看效果再加 |
| 效果好（ACOS 下降或持平、出单增加） | +50% | 7 天 | — |
| 持续好 | +75% ~ +100% | — | 一般不超过 +100% |
| 效果变差 | 退回上一档 | — | — |

```
TOS 加价纪律：
1. 起步永远 +25%，不许直接 +50% 以上
2. 每次调整后至少观察 7 天（含一个完整周末）
3. ACOS 没改善就退档，不许"再坚持一下"
4. 加价幅度 ≥ +100% 要警惕：可能你的基础出价本身就偏低，该调的是基础出价而不是 TOS 加价
```

### 第二步：分时调价——先摸清你的"成交时刻表"

分时调价的前提是**知道你的买家什么时候买**。不同品类差异巨大：

| 品类 | 典型下单高峰（站点当地时间） | 典型低谷 |
|---|---|---|
| 母婴 / 家居 | 20:00-23:00（家长哄睡后） | 03:00-07:00 |
| 办公用品（B2B） | 09:00-12:00（工作时段） | 22:00-06:00 |
| 礼品类 | 12:00-14:00 + 20:00-22:00 | 03:00-08:00 |
| 美妆个护 | 07:00-09:00 + 21:00-23:00 | 02:00-06:00 |

但这是"经验值"，**你必须用自己的数据验证**。怎么验证：

```
摸清成交时刻表（每月做一次）：
1. 下载过去 30 天的「按日期」订单报表（Business Reports）
2. 按"小时"维度拆分订单量（站点当地时间）
3. 算出每个时段的订单占比
4. 标出三档：
   - 高峰时段（订单占比 ≥ 6%/小时）
   - 平峰时段（3%-6%/小时）
   - 低谷时段（< 3%/小时）
```

### 第三步：分时调价的执行规则

亚马逊 SP 广告后台**原生不支持按小时调价**，但有三种替代方案：

| 方案 | 做法 | 适用 |
|---|---|---|
| **A. 预算分配法**（最简单） | 给广告活动设一个较低日预算，让钱集中在高峰花完 | 新手、预算小 |
| **B. 多活动分时段法** | 建两个活动：高峰活动（高竞价+充足预算）、平峰活动（低竞价） | 中阶、预算够 |
| **C. 第三方工具调价** | 用软件按小时自动调竞价（Sellozo / Perpetua 等） | 高阶、多 SKU |

新手推荐 **方案 A**，规则如下：

```
低谷时段（占比 < 3%/小时）：
  → 不刻意降价，但要确保日预算不会被低谷"偷"光
  → 检查：早上 8 点（站点时间）看预算消耗，如果消耗 > 40%，说明钱被低谷偷走了

高峰时段（占比 ≥ 6%/小时）：
  → 确保预算够撑到高峰，必要时提高日预算
  → TOS 加价在高峰前 1 小时生效（系统有延迟）
```

> 关键认知：**分时调价的核心不是"低谷降价"，而是"确保预算在高峰花得出去"。** 很多人预算被凌晨的垃圾流量耗光，导致晚上真正的高峰没钱跑——这才是预算效率的头号杀手。

### 日预算规则的红线

亚马逊现在的日预算规则必须遵守，否则活动会被暂停或限制：

| 规则 | 数字 | 违反后果 |
|---|---|---|
| 日预算最低下限 | 一般 $1（部分站点 €1） | 设低了活动不跑 |
| 日预算超出月度限额 | 不超过月度上限的 100% | 系统暂停活动 |
| 预算花完 | 当天不再展示 | 第二天自动恢复 |

> 分时调价不能违反日预算下限。想用"低谷降预算到 0"来实现分时？不行——预算为 0 活动就停了。正确做法是方案 B：用多个活动分时段跑。

## 实操要点

- **先看 Placement 报表再决定抢不抢 TOS**：TOS CVR 是整体 1.3 倍 + CTR 是 1.5 倍才抢，不达标就别抢。
- **TOS 加价永远从 +25% 起步**：每次调整观察 7 天，ACOS 没改善就退档，不许硬扛。
- **每月摸一次"成交时刻表"**：用过去 30 天订单数据按小时拆分，标出高峰/平峰/低谷三档。
- **预算分配比调出价更重要**：分时调价的核心是"让钱在高峰花出去"，不是"低谷少花钱"。
- **日预算下限不能破**：所有调价动作在平台规则框架内做，不要用违规手段卡时段。

## 动手练习

挑你一个跑得最久、数据最稳的 SP 活动做一次完整诊断：(1) 下载 Placement 报表，算 TOS 的 CVR 倍数和 CTR 倍数，判断该不该抢 TOS；(2) 下载过去 30 天按小时拆分的订单数据，画出你的"成交时刻表"，标出高峰/平峰/低谷三档；(3) 检查你最近一周的预算消耗节奏——每天早上 8 点（站点时间）预算已经消耗了多少？如果超过 40%，说明钱被低谷偷走了，需要调预算分配。
