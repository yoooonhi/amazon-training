# 第2节 · ASIN 定位与流量卡位

> **手册：[广告打法手册](./)**
> **本节收录**：✅ ASIN 定位四组法、✅ 类目定位、✅ 广告霸屏/防御
> **本节目标**：掌握 ASIN 定位的四组分类框架（本手册最有含金量的部分），知道每组投什么 ASIN、用什么竞价策略。

## 这节解决什么问题

很多人开 ASIN 定位广告就一股脑塞竞品 ASIN，结果转化参差不齐、预算乱花。其实**ASIN 定位不是"投竞品"，而是"按关系分类投"**——不同关系的 ASIN，应该用完全不同的策略：

- **直接竞品**（你想抢它的买家）→ 用优势定位，敢出价
- **弱竞品**（评分低、价格高、图差的）→ 用捡漏定位，低竞价蹭
- **互补品**（买家会一起买的）→ 用互补定位，五五开机会
- **自己店铺的同类**（防止流量外流）→ 流量闭合组，把买家圈在自己店里

把这四种关系分清楚，分别建组、分别配竞价策略，转化效率会显著提升。

> 关键认知：**ASIN 定位四组法是本手册最有含金量的部分**。学会它，你的 ASIN 定位广告效率能甩开 80% 的卖家。

## 核心知识

### ✅ 打法五：ASIN 定位四组法

ASIN 定位（Product Targeting）允许你**主动选择**广告展示在哪些 ASIN 的详情页上。相比关键词广告的"被动匹配"，ASIN 定位有更多主动权——你能针对性地挑有利 ASIN 蹭流量、抢流量。

**第一层：ASIN 选择——开 4 个独立广告组**

```
四组分类（按 ASIN 与你的关系）：

组1 捡漏定位组：选「比你弱」的 ASIN
   → 关键词自然排名 + 广告排名前 4 的 ASIN
   → 品牌分析里的「替代购买 ASIN」
   → 这些 ASIN 的买家会看几个备选，你蹭到就有机会

组2 优势定位组：选「你有优势」的 ASIN
   → 广告报告里 ACOS 表现好的竞品 ASIN
   → 用工具筛选：Review 少 / 评分低 / 价格高 / 主图质量差的 ASIN
   → 你的产品在这些维度上明显更强，转化率会高

组3 互补定位组：选「会被一起买」的 ASIN
   → 品牌分析里的「商品比较 ASIN」
   → 商品页面关联位（ Sponsored related / FBT）前 4 ASIN
   → Helium10 Black Box → Product Targeting 功能
   → 筛 Frequently Bought Together / Customers Also Bought

组4 流量闭合组：投自己店铺的同类/互补 ASIN
   → 直接定位自己店铺的互补、同类 ASIN
   → 形成流量闭环，买家在你店里转，不外流给竞品
```

**ASIN 来源速查**：

| 来源 | 用途 | 工具 |
|---|---|---|
| 关键词自然排名 + 广告排名前 4 | 捡漏定位 | 前台搜索 + 工具反查 |
| 品牌分析的「替代购买 ASIN」 | 捡漏定位 | Brand Analytics（品牌备案后） |
| 广告报告 ACOS 表现好的竞品 | 优势定位 | Search Term Report |
| Review 少/评分低/价格高/主图差的竞品 | 优势定位 | 卖家精灵/鸥鹭/Helium10/JS/Sif |
| 品牌分析的「商品比较 ASIN」 | 互补定位 | Brand Analytics |
| FBT / Customers Also Bought | 互补定位 | 前台详情页 + Helium10 |
| 自己店铺同类/互补 ASIN | 流量闭合 | 卖家后台 |

> 实操要点：互补组选 ASIN 时**一定要筛"有价格和 Review 优势"的**——互补品的买家心智不锁定单一品牌，你只要价格/评分对得上，就有转化机会。

---

**第二层：竞价策略——四组各配不同策略**

| 广告组 | 关系 | 竞价策略 | Bid 设置 |
|---|---|---|---|
| 捡漏定位组 | 对方比你强 | 动态竞价-提高和降低 | 较低 bid + 商品页竞价浮动百分比，升至历史出单 CPC 加权平均 |
| 优势定位组 | 你比对方强 | 固定竞价 | bid = CPC 加权平均值 |
| 互补定位组 | 五五开机会 | 动态竞价-只降低 | bid = CPC 加权平均值 |
| 流量闭合组 | 自己店铺 | 动态竞价-只降低 | bid = CPC 加权平均值 |

**为什么这样配**：

```
捡漏组：对方比你强，硬抢会亏
  → 用「提高和降低」让系统只在有转化机会时才加价
  → 设低基座竞价 + 高位置溢价，专挑便宜坑位

优势组：你比对方强，转化率高
  → 用「固定竞价」稳定占位
  → bid 给到 CPC 加权平均，确保竞争力

互补组：机会五五开
  → 用「只降低」控制成本
  → 系统判断能出单就跑，不出单自动降竞价

流量闭合组：自己店铺内的流量流转
  → 用「只降低」防止自我竞价抬价
  → 目的不是出单，是把买家留在自己店里
```

> 红线提醒：**流量闭合组不要用"提高和降低"**——你定位的是自己店铺的 ASIN，系统会判断有转化机会而不断加价，结果就是自己跟自己竞价、CPC 越来越高。

### ✅ 打法六：类目定位（Category Targeting）

**底层逻辑**：类目定位是把广告投给整个类目下的所有 ASIN。**流量极泛、转化极差**——大部分有转化诉求的卖家都不推荐做，性价比太低。

但如果你想试，唯一正确的姿势是**用好 Refine 功能**对 ASIN 进行精细化筛选：

```
类目定位 + Refine 筛选的正确做法：

① 选定一个相关类目（与你产品二级类目关联度高的）
② 点 Refine（细化）按钮，按以下维度筛选：
   - 价格区间（避开价格带外的 ASIN）
   - 星级（避开评分差异太大的 ASIN）
   - 品牌（排除某些品牌）
   - Prime 配送状态
③ 把广告范围缩小到「真正可能转化的 ASIN 子集」

不筛 Refine 直接投整个类目 = 流量黑洞，预算烧光没转化
```

> 实操要点：类目定位**只适合特定场景**——你的产品是配件类（钓鱼配件、运动配件等）、关联性极强的场景类（室内运动场景）。主推品别用类目定位，转化率会让你怀疑人生。

### ✅ 打法七：广告霸屏 / 流量防御策略

**底层逻辑**：让自己产品占据大多数广告坑位，**尤其在自己卖得好的产品详情页下方**——不让竞品占领这些坑位、分走流量。

**适用判断**：
- ✅ 垂直类目（同一店铺有多条同类产品线）
- ✅ 多变体产品（颜色/尺寸/款式丰富）
- ✅ 交叉店铺有同类产品
- ❌ 单 SKU、单一产品线的小卖家（霸屏玩不起来）

```
霸屏策略的具体步骤：

1. N 多个 SKU 投一组词
   → 多个变体共享关键词流量

2. N 多个 SKU 投一个 ASIN
   → 多个变体都出现在某个竞品详情页

3. 广告结构：
   1 个广告活动 (campaign) + 1 个广告组 (ad group)
   + 选多个相似 SKU + 1 个关键词或 1 个 ASIN
   → 如果投多个关键词，要有几个 SKU 都有比较好的标签

4. 组合 SP + SB + SD 三种广告
   → SP 占搜索结果和详情页
   → SB 占品牌位
   → SD 占重定向和展示位
   → 三管齐下覆盖更多坑位

5. 自动广告做关联
   → 自己店铺同类/互补品开一个自动广告
   → 把 SKU 全放进去，让产品在广告上形成关联
   → 用固定竞价
```

**关键词独立建活动**：

> 关键铁律：**branded keyword（品牌关键词）和 non-branded keywords（非品牌关键词）必须独立创建广告活动**，方便分别分配预算和调整竞价。混在一起会导致预算分配混乱、看不出品牌词和非品牌词各自的贡献。

> 实操要点：霸屏策略的本质是**"流量闭环"**——把买家从搜索框到详情页到加购，每一步都尽量留在自己店铺。中小卖家做不了霸屏，但**流量闭合组**（投自己店铺互补品）谁都能做，是低门槛版的霸屏。

## 实操要点

- **ASIN 定位四组法是核心**：捡漏/优势/互补/流量闭合四组要分开建、分开配竞价策略，不要混在一起。
- **竞价策略别配错组**：捡漏组用动态-提高降低（避免硬抢）、优势组用固定（稳定占位）、互补组用只降低（控成本）、闭合组用只降低（防自我抬价）。
- **类目定位必须用 Refine**：不筛 Refine 直接投整个类目 = 流量黑洞。只适合配件类/场景类，主推品别用。
- **霸屏只适合垂直大卖**：中小卖家玩不了，但"流量闭合组"是低门槛版霸屏，谁都能做。
- **品牌词和非品牌词独立建活动**：方便看各自贡献，混在一起预算分配会混乱。

## 动手练习

选一个有自然单的产品，按四组法搭一套 ASIN 定位广告：
1. **捡漏组**：找出 10 个关键词排名前 4 的 ASIN + 5 个品牌分析替代购买 ASIN，建组用动态-提高降低
2. **优势组**：用工具筛 10 个 Review 少 / 评分低 / 价格高 / 主图差的竞品 ASIN，建组用固定竞价
3. **互补组**：找出 10 个 FBT / Customers Also Bought ASIN，建组用只降低
4. **流量闭合组**：把自己店铺的同类/互补 ASIN 放进去，用只降低

跑 14 天后，对比四组的 ACOS 和转化率，看哪组最值得加预算。重点练的是**用关系分类驱动策略选择**，而不是一股脑塞竞品 ASIN。
