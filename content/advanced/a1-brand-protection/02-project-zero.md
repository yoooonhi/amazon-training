# 高级1.2 Project Zero 与自动保护

> **高级1：品牌保护与长期主义**
> **本课目标**：能用 Project Zero 三件套（自动保护、仿冒自助下架、商品序列化）组合打击假货与跟卖，并知道它的边界与风险。

## 这节课解决什么问题

上节课的 Transparency 解决的是"防止没有码的货混入 FBA"，它事前防御很强，但有滞后性——你得先把所有库存都贴码、新货持续贴码，整套跑通要时间。**在 Transparency 还没铺开的窗口期，或者遇到 Transparency 防不了的 FBM/VC 跟卖，你怎么办？**

中级课教过品牌备案里的"举报违规行为（Report a Violation）"，但那是手动提交、等审核，一个投诉排队几天到几周，遇到批量假货根本打不过来。

Project Zero（零容忍计划）是亚马逊给品牌方的**三件套武器库**，其中最值钱的是"自助下架"——你自己点一下，售假者的 Listing 就没了，不用等亚马逊审核。这节课讲清楚三件套各自干什么、怎么组合、以及"自助下假架"的**滥用风险**（这是双刃剑，用不好反伤自己）。

## 核心知识

### Project Zero 三件套

| 工具 | 作用 | 力度 | 谁触发 |
|---|---|---|---|
| **① 自动保护（Automated Protections）** | 用你的品牌数据（商标+上传的商品图样）机器扫描全站，自动撤下疑似假货 | 主动、持续、机器24h扫 | 系统自动 |
| **② 自助假冒商品移除（Self-service Counterfeit Removal）** | 你发现假货/侵权，一键下架，无需等亚马逊审核 | 即时、杀伤大 | 你手动 |
| **③ 商品序列化（Product Serialization）** | 就是上节课的 Transparency 一物一码 | 事前防御 | 工厂贴码 |

> 关键认知：**Project Zero 不是"第四个工具"，它是一个套餐，把自动扫描+自助下架+Transparency 打包给品牌方。** 你可以理解为：Transparency 是事前门禁，自动保护是 24h 监控摄像头，自助下架是你自己的红章删除权。三者配合才是完整防御。

### 自动保护的工作原理

自动保护不是你手动设规则，而是**亚马逊用你提交的品牌资产训练机器模型，然后扫全站**：

```
你提交给亚马逊的「品牌资产」：
  - 注册商标图样（文字+图形）
  - 商标注册号和覆盖类别
  - 你正品的产品图、logo、包装图
  - 你授权的正规经销商清单（可选）

亚马逊训练模型后自动扫描：
  → 全站所有 Listing 的标题/五点/图片
  → 发现「疑似用了你的商标/logo但不是你的 Listing」
  → 自动判断是否为假货/侵权
  → 命中 → 自动撤下，并通知你
```

> 关键纪律：**自动保护的好坏取决于你提交的品牌资产质量。** 你上传的正品图样越全、授权渠道越清晰，机器扫得越准。提交得糊弄，自动保护就误判连连。

### 自助下架：最锋利也最危险的武器

自助下架是 Project Zero 最被卖家"垂涎"的功能——**你点一下，对方的 Listing 就没了，不用排队等审核。** 但这是双刃剑：

| 优势 | 风险 |
|---|---|
| 即时生效，几小时内售假者下架 | **滥用会被反噬**：恶意下架竞争对手会触发对你的调查 |
| 不用攒证据等审核，效率极高 | 下错了要担责，对方申诉成功你账号受影响 |
| 批量处理能力强 | 亚马逊监控你的下架准确率，准确率低会被踢出计划 |

> 红线警告：**自助下架只能用于「确凿的假货和侵权」，绝不能用来打击合法竞品或正常跟卖。** 亚马逊会统计你每次下架后对方是否申诉成功——如果申诉成功率高（说明你下错了），你的 Project Zero 资格会被撤销，甚至触发品牌备案调查。把这当"消灭竞品"工具的人，最后都把自己玩死了。

#### 自助下架的标准操作（SOP）

```
触发条件确认（下架前必查）：
  ☐ 对方卖的是「假货」或「未授权用你的商标/logo」——不是单纯价格低
  ☐ 你已截图存证（Listing页、对方卖家信息、侵权点特写）
  ☐ 你确认这不在「合法跟卖/分销」范畴

执行：
  1. 进入 Project Zero 后台 → 输入侵权 ASIN 或跟卖者信息
  2. 选择侵权类型（假冒商标/假冒商品/版权侵权）
  3. 填写证据描述（具体到哪张图哪个词侵犯了什么）
  4. 提交 → 通常几小时内下架

存档（必做）：
  - 保存每次下架的截图、证据、时间
  - 跟踪对方是否申诉，申诉结果
  - 维护「下架准确率」自查表：成功下架/申诉翻盘的比例
```

### Project Zero vs Transparency vs 品牌备案投诉：怎么选

新手最常困惑"我到底该用哪个"。决策逻辑：

| 场景 | 首选工具 |
|---|---|
| 想从源头阻止 FBA 跟卖/假货进仓 | **Transparency**（事前门禁） |
| 发现明确的假货，要立刻下架 | **Project Zero 自助下架**（即时） |
| 持续监控全站有没有人冒用你商标 | **Project Zero 自动保护**（机器扫） |
| 一般性违规（非假货，如图片盗用/文案抄袭） | **品牌备案 Report a Violation**（等审核） |
| 跟卖是合法分销/授权商，只是价格乱 | **不是工具问题，是渠道管理问题** |

> 关键认知：**Transparency 和 Project Zero 不是二选一，而是组合拳。** 最佳实践是 Transparency 铺底（防 FBA 进仓）+ 自动保护监控 + 自助下架做"快速反应部队"。

### 申请 Project Zero 的门槛

比 Transparency 更高：

| 条件 | 要求 |
|---|---|
| 品牌备案 | 必须完成 |
| 政府注册商标 | 必须有（商标 pending 不行） |
| 历史投诉记录 | 有过成功的假冒商品举报记录（用过 Report a Violation 且成功过） |
| 政府商标号 | 需提交有效的政府商标注册号 |

> 门槛提醒：**如果你从没用品牌备案投诉过假货，Project Zero 基本进不去。** 它要求你先证明"你有识别和打击假货的能力"。所以路径是：先在品牌备案里成功举报几次 → 再申请 Project Zero。

## 实操要点

- **三件套是组合拳不是单选**：Transparency 防 FBA 进仓、自动保护监控全站、自助下架快速反应——三个都开才叫完整防御。
- **自动保护要喂数据**：提交的正品图样、授权渠道清单越全，机器扫得越准，别糊弄。
- **自助下架是核武器，慎用**：只打确凿假货，绝不用来打合法竞品。维护你的下架准确率，滥用会被踢出甚至反噬。
- **先投诉积累再申请**：Project Zero 要看历史假冒举报成功记录，路径是"品牌备案举报→成功若干次→再申请Zero"。
- **下架后必存档跟踪**：每次自助下架都记录证据、跟踪对方申诉结果，定期自查准确率。
