# 高级1.1 Transparency 透明计划

> **高级1：品牌保护与长期主义**
> **本课目标**：能判断自己的产品该不该上 Transparency，算清成本与防御收益，并走通申请到上线的完整流程。

## 这节课解决什么问题

中级课你学了品牌备案、A+、品牌故事，能把"品牌感"做出来。但一旦你的 Listing 起量、上了首页，**第一波来吃你的不是竞品，而是跟卖**。一夜之间 Buy Box 被抢、价格被压到你的成本线、评论区被灌差评——你在前面冲锋，后面有人在拆台。

很多运营的应对是"被跟卖了去服务商买个赶跟卖软件"。这种打法有三个致命问题：一是**治标不治本**，赶走一波还有下一波；二是**有合规风险**，部分粗暴手段本身可能触发账号警告；三是**永远被动**，你永远在"发现跟卖→处理"的循环里耗。

Transparency（透明计划）是亚马逊官方给品牌卖家的"**从源头杜绝跟卖**"的工具。这节课讲清楚它的原理、成本、申请门槛，以及最关键的——**它到底能防什么、防不了什么**，别花冤枉钱还以为买了金钟罩。

## 核心知识

### Transparency 的底层原理：一物一码

跟卖能存在的根源是"亚马逊把同一个 ASIN 下所有卖家的货视为可互换"。Transparency 的做法是：**品牌方给每一件出厂商品贴一个独一无二的二维码（Transparency code）**，亚马逊入库和出库时都扫码验证。

```
Transparency 工作机制：
  品牌方 → 从亚马逊申请一批 Transparency code（每件1个，唯一）
       → 在工厂生产环节把码贴到每个单品包装上
  亚马逊FBA收货 → 扫码验证 → 码对，入库；码不对或无码，拒收/销毁
  亚马逊出库给买家前 → 再扫一次码 → 码对才发出
  其他卖家想跟卖这个ASIN → 他必须也能提供有效 Transparency code → 他没有 → 无法跟卖
```

> 关键认知：**Transparency 不是"赶走跟卖"，而是"让跟卖根本无法上架你的 ASIN"。** 这是事前防御，不是事后补救。一旦你的 ASIN 开启了 Transparency，没有你的码的货，连 FBA 都进不了。

### 申请门槛：不是所有卖家都能进

| 条件 | 要求 | 说明 |
|---|---|---|
| 品牌备案 | 必须已完成 Amazon Brand Registry | 且商标是"活的有效商标" |
| 商品标识 | 每个单品需有 GTIN（UPC/EAN） | 且能通过 GS1 验证 |
| 包装条件 | 商品有实体包装，能贴码 | 散装、无独立包装的做不到 |
| 能力 | 能在供应链源头（工厂）贴码 | 你能控制生产环节 |
| 地区 | 按站点分别申请 | 美/欧/日/澳各自独立 |

> 红线提醒：**没有品牌备案、或商标还在申请中（pending）的，直接pass，先去解决品牌备案（中级3）。** Transparency 是品牌备案之后的进阶工具，顺序不能反。

### 成本结构：按件计费，不是一次性费用

Transparency 的成本是**每个 Transparency code 一个固定费用**（按申请地区不同，约 $0.01-$0.05/个），没有月费。但真正的隐性成本在贴码环节：

| 成本项 | 金额/说明 | 是否可省 |
|---|---|---|
| 每个 code 费用 | 约 $0.01-$0.05/件（按地区） | 不可省，亚马逊收 |
| 贴码人工/标签 | 工厂代贴，约 ¥0.1-0.3/件 | 可谈判分摊 |
| 最小起订量 | 单次申请有 MOQ（如 1 万个起） | 影响资金占用 |
| 系统对接 | 量大时需对接工厂打印系统 | 量小可手动 |

```
成本测算示例：
  产品A，月销 2000 件，售价 $25
  Transparency 码费：$0.02/件 × 2000 = $40/月
  贴码成本：¥0.2/件 × 2000 ≈ $56/月
  月总成本 ≈ $96，单件分摊约 $0.048

  判断逻辑：
  - 如果跟卖已造成你月损失 > $96（降价保Buy Box / 被抢销量）→ 划算
  - 如果产品从没被跟卖、客单价低、毛利薄 → 可能不划算
  - 如果产品是高客单、易被仿、品牌核心SKU → 强烈建议上
```

### 防御效果：能防什么、防不了什么

这是最容易被误解的部分。很多卖家以为上了 Transparency 就万事大吉，**不是的**。

| 跟卖类型 | Transparency 能否防御 | 说明 |
|---|---|---|
| FBA 跟卖（同款货） | 能 | 跟卖者的货进不了FBA（无码），无法跟卖 |
| FBM 自发货跟卖 | **不一定** | 部分情况跟卖者可绕过（需额外设置/申诉） |
| 假货跟卖（仿品） | 能 | 假货没有有效码 |
| 恶意篡改listing后被跟卖 | **不能** | 这是listing被劫持问题，先解决劫持 |
| VC 账号/亚马逊自营跟卖 | **基本不能** | 平台自有渠道，Transparency 约束力弱 |

> 红线警告：**Transparency 防 FBA 跟卖很有效，但它不是 100% 防弹。** 特别要警惕"FBM 跟卖绕过"和"VC 账号跟卖"——这两种需要配合 Project Zero（下节课）和其他手段。

### 上线完整流程

```
1. 确认品牌备案已完成且商标有效
2. 进入 transparency.amazon.com 提交申请
3. 亚马逊审核（1-2周），通过后获得后台
4. 在后台为每个 ASIN 申请 Transparency code
5. 下载码文件 → 发给工厂
6. 工厂在包装环节贴码（每个单品1个）
7. 货物发往FBA → 亚马逊扫码验证入库
8. 开启"主动保护"：无码货无法跟卖该ASIN
9. 持续：每批新货都要贴码，否则新批次进不了FBA
```

> 关键纪律：**上了 Transparency 就不能停。** 一旦停贴码，新批次货物发不进 FBA，等于自断货源。所以上之前要算清长期成本，不能"先试试"。

## 实操要点

- **先算账再上**：用"跟卖历史损失 vs Transparency 月成本"做决策，别为了"安全感"无脑上，低毛利跑量款可能不划算。
- **必须能控制生产环节**：你做不到让工厂贴码，Transparency 就用不起来——这是硬门槛，1688 现货代发的玩法基本无缘。
- **按站点分别申请**：美国 Transparency 不自动覆盖欧洲，多站点卖家每个目标市场都要单独走流程。
- **警惕 FBM 跟卖绕过**：开启 Transparency 后，定期巡检 ASIN 是否仍有 FBM 跟卖出现，发现立即用 Project Zero 投诉。
- **新批次必须持续贴码**：停贴码=断货风险，把"申请码+工厂贴码"固化进采购SOP，不能漏。
