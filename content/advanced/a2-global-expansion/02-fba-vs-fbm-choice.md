# 高级2.2 自发货 vs FBA 选择

> **高级2：多站点全球扩张**
> **本课目标**：能在远程配送（NARF/EF）、FBA本地发货、海外仓+FBM 三种模式中，按场景做正确的履约决策，不踩"一招打天下"的坑。

## 这节课解决什么问题

上节课你学会了"先选哪个站点"。但选定站点后，下一个决策是**怎么发货**——是用本地FBA、用远程配送从主战场发、还是用海外仓+自发货（FBM）？这个决策直接决定你的利润、时效、合规复杂度和买家体验。

很多卖家的做法是"美国怎么做欧洲就怎么做"——美国用FBA，欧洲也全压FBA。结果有的产品在欧洲压了一仓库存卖不动（FBA滞销费吃光利润）；有的产品明明适合远程配送试水，却一次性备了3个月货，结果选品没验证成功库存砸手里；有的高客单大件非要塞FBA，费用贵到没法卖。

问题在于：**履约模式不是"哪个好"，是"哪个场景适合哪个"。** 不同产品（体积/客单/销量稳定性）、不同阶段（试水/放量/成熟）、不同站点（远近/合规），最优履约方式完全不同。这节课把三种主流模式的适用场景和决策框架讲清楚。

## 核心知识

### 三种主流履约模式

| 模式 | 怎么运作 | 时效 | 成本 | 控制力 |
|---|---|---|---|---|
| **① 本地FBA** | 货先发到目标国FBA仓，买家下单亚马逊本地配送 | 快（1-2天） | 中（FBA费+头程+仓储） | 低（亚马逊管） |
| **② 远程配送（NARF/EF）** | 货在主战场FBA仓，跨境配送给目标国买家 | 慢（5-12天） | 中低（配送费+可能关税） | 低 |
| **③ 海外仓+FBM** | 货发到目标国第三方海外仓，卖家自发货（FBM/MFN） | 中（2-5天） | 视情况（海外仓费+尾程） | 高（卖家管） |

> 关键认知：**没有"最好"的模式，只有"最适合当前场景"的模式。** 新市场试水用远程配送，跑通了转本地FBA，超大件或特殊品考虑海外仓。一招打天下必然踩坑。

### 模式一：本地FBA（最主流，但有代价）

这是绝大多数卖家的默认选择，优势明显：Prime标志、Buy Box权重高、亚马逊管物流省心。但代价：

```
本地FBA的「隐藏代价」：
  - 必须先压货到目标国FBA仓（资金占用 + 备货风险）
  - 选品没验证成功 → 库存滞销 → 长期仓储费
  - 大件/超重 → FBA费飙升，可能没法卖
  - 跨国FBA（如欧洲EF）涉及多国，仓储分配复杂
  - 退货回到目标国FBA仓，处理成本高

适合本地FBA的场景：
  ✓ 选品已在主战场验证成功，有销量数据
  ✓ 产品体积/重量在FBA经济档内
  ✓ 销量稳定可预测，备货风险可控
  ✓ 你愿意接受资金占用换Prime权重
```

> 红线提醒：**没验证选品就压本地FBA，是多站点扩张最大的库存杀手。** 很多卖家欧洲压了一仓卖不动，长期仓储费+滞销，比没扩张还亏。新市场先用远程配送试水。

### 模式二：远程配送（试水神器，但有上限）

远程配送让你**用主战场的FBA库存，跨境卖给目标国买家**，不用提前压货到目标国。这是新市场试水的最佳工具：

| 远程配送类型 | 覆盖 | 运作 |
|---|---|---|
| **NARF（北美远程配送）** | 美国FBA库存 → 加拿大/墨西哥买家 | 美国卖家一键开通 |
| **EF（欧洲配送网络）** | 某国FBA库存 → 其他欧洲国买家 | 需欧洲多国布局 |

```
远程配送的优势（为什么是试水神器）：
  - 不用提前压货到目标国 → 零备货风险
  - 用现有FBA库存 → 资金不额外占用
  - 能快速测出「这个产品在目标国有没有需求」
  - 一键开通，操作简单

远程配送的局限（为什么不能长期依赖）：
  - 时效慢（5-12天），无Prime次日达体验，转化率偏低
  - 配送费较高，利润被吃
  - 买家可能承担进口关税（影响体验和转化）
  - 目标国买家更倾向本地FBA的快速到货
  - 销量起来后，远程配送的成本和体验劣势放大
```

> 实操纪律：**远程配送是「试水」工具，不是「长期」方案。** 标准节奏：远程配送测出有需求 → 销量稳定到一定量级 → 转本地FBA提时效和转化。长期靠远程配送会让你的转化率和利润都吃亏。

### 模式三：海外仓+FBM（特殊场景的利器）

海外仓（第三方海外仓储）+ FBM（卖家自发货），是你把货发到目标国的第三方仓，自己管发货。控制力最强但最费精力：

```
海外仓+FBM适合的特殊场景：
  ✓ 超大件/超重货 → FBA费太贵或FBA不收，只能海外仓
  ✓ 高客单+低周转 → 长期FBA仓储费不划算
  ✓ 需要定制/组合/贴标等增值服务 → FBA做不了
  ✓ 多平台共用库存（亚马逊+独立站+其他平台）
  ✓ FBA经常断货/限制入仓时的备选

海外仓+FBM的代价：
  - 没有Prime标志（除非参加SFP项目），Buy Box权重低
  - 卖家自管物流，时效/退货/客服全靠自己
  - 对海外仓依赖高，仓的服务质量影响你的评价
  - 合规（VAT/税务）自己处理，不享受FBA的某些便利
```

> 红线提醒：**FBM最大的代价是「没有Prime权重」。** 除非产品属性逼着你用（超大件、定制），否则同款产品FBA通常比FBM转化率高、排名好。别为了"控制力"放弃Prime，得不偿失。

### 三模式决策框架

```
按产品阶段 + 产品属性 决策：

【新市场试水期】（销量未知）
  → 首选：远程配送（NARF/EF）
  → 理由：零备货风险，快速测需求
  → 验证标准：连续4周有稳定出单 + 转化率达标

【验证成功放量期】（销量已起来）
  → 首选：本地FBA
  → 理由：提时效和转化，拿Prime权重
  → 备货量：参考主战场销量×调整系数

【特殊产品】（FBA不经济或不收）
  → 大件/超重 → 海外仓+FBM
  → 高客单低周转 → 海外仓+FBM
  → 需定制/组合 → 海外仓+FBM
  → 理由：FBA费吃光利润或FBA根本不收

【FBA限制/断货期】（临时）
  → 备选：海外仓+FBM过渡
  → 理由：保供货不断，但回归FBA优先
```

### 决策矩阵（一图选定）

| 场景 | 推荐 |
|---|---|
| 新市场，没销量数据 | 远程配送试水 |
| 新市场，远程配送测出稳定需求 | 转本地FBA |
| 老市场，标准品，销量稳定 | 本地FBA |
| 超大件/超重 | 海外仓+FBM |
| 高客单低周转大件 | 海外仓+FBM |
| 需定制/组合/贴标 | 海外仓+FBM |
| FBA入仓受限/断货 | 海外仓+FBM过渡，恢复后回FBA |
| 多平台共用库存 | 海外仓+FBM |

> 关键认知：**同一批货可能混合用多种模式**——比如主力SKU走FBA，试水SKU走远程配送，超大件走海外仓。按SKU属性和阶段分别决策，不是一刀切。

### 成本测算示例（决策的数字依据）

```
产品X，美国验证成功，准备进德国：

方案A：远程配送（EF从某国仓发德国）
  售价€20，配送费€5，利润€6 → 净利率30%
  但时效7-10天，转化率预估比FBA低30%
  → 实际月单量 = FBA预估 × 70%

方案B：本地FBA（压货到德国仓）
  售价€20，FBA费€4.5，利润€7 → 净利率35%
  时效1-2天，转化率高
  但需压200件库存（资金€X）+ 备货风险

方案C：海外仓+FBM
  售价€20，海外仓费+尾程€6，利润€5 → 净利率25%
  无Prime，转化率最低，但适合大件

决策：
  试水期（前2个月）→ 方案A远程配送，零风险测需求
  验证成功（月单>50）→ 方案B本地FBA，提转化
  若X是大件 → 直接方案C海外仓
```

## 实操要点

- **新市场先用远程配送试水**：零备货风险测需求，验证成功再转本地FBA，别没数据就压一仓货。
- **FBA是转化率和Prime权重的主流**：标准品稳定销量优先FBA，别为了"控制力"放弃Prime。
- **海外仓只用于FBA不经济或不收的场景**：超大件、定制、高客单低周转、多平台共用，别滥用。
- **按SKU分别决策**：同一批货可能主力走FBA、试水走远程、大件走海外仓，不一刀切。
- **用数字决策**：测算三种模式的单件净利和转化率影响，别凭感觉选履约方式。
