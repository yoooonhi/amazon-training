# 1.6 买家消息边界

> **模块1：平台认知与账号安全**
> **本课目标**：搞清楚和买家沟通时"能说什么、不能说什么"，避免因为一句不当的话被判定操纵评论或违规。

## 这节课解决什么问题

亚马逊允许卖家通过站内信（Buyer-Seller Messaging）回复买家咨询。但很多新手在沟通时"好心办坏事"——为了好评多说两句、为了售后加个微信——结果一句不当的话就把账号搭进去了。这节课帮你画清沟通的边界。

## 核心知识

### 你能做的事（合规）

| 场景 | 你可以做的 |
|---|---|
| 买家问产品问题 | 详细、诚实地回答（尺寸、材质、用法等） |
| 买家要退货 | 指导退货流程，必要时直接退款 |
| 买家反映使用问题 | 提供使用说明、 troubleshooting 建议 |
| 买家抱怨物流 | 解释物流状态，必要时补发或退款 |
| 订单异常需要联系买家 | 通过站内信主动说明情况 |

### 你不能做的事（红线）

| ❌ 红线行为 | 例子 |
|---|---|
| 在消息里要好评 | "如果满意请留个好评哦~" |
| 用折扣/返现换好评 | "留好评返你 30%""加微信有优惠" |
| 引导到站外交易 | "我们的官网更便宜""加我微信下单" |
| 暗示好评 | "希望您满意，期待您的反馈""您的好评对我们很重要" |
| 频繁骚扰买家 | 同一个问题反复发消息催 |
| 索要买家联系方式 | "能加您微信吗""留个电话方便联系" |
| 要求删评/改评 | "看到您留了差评，能否删除，我们退款给您" |

### 一个核心原则

> **站内信的唯一目的，是解决买家的问题。**

不要借机做营销、不要提评论、不要引流到站外。把每次沟通都当作"纯粹的服务"，这样最安全。

### 回复时间有要求

- 亚马逊要求卖家在收到买家消息后 **24 小时内**回复（包括周末和节假日）。
- 超时不回复，会影响"联系回复时间"指标，长期不达标会影响账号健康。
- 即使你暂时解决不了，也要先回一句"我们已收到，正在处理"，避免超时。

### 这些情况要特别小心

1. **买家主动提到要好评**：你也不能顺着说"谢谢您的好评"，直接感谢购买即可，不要回应评论话题。
2. **买家主动留了站外联系方式**：即使买家给了微信，你也不要加——通过站外沟通出了问题，亚马逊不保护你。
3. **买家威胁"不解决就差评"**：不要被要挟做违规操作（如无条件高额退款）。可以通过站内信礼貌沟通，必要时联系卖家支持。
4. **群发营销邮件**：禁止给所有买家群发促销/新品邮件，除非买家主动订阅。

## 实操要点

- **建立消息模板**：准备几套标准的回复模板（退货流程、产品使用说明、物流查询），既快又合规。
- **回复前先自检**：发消息前问自己——这句话里有没有"好评""微信""优惠"这些敏感词？有就改掉。
- **用站内信，不用私人邮箱**：即使买家留了邮箱，也只在亚马逊站内信系统回复，所有沟通都有记录。
- **保留沟通记录**：重要的沟通（尤其涉及退款、纠纷）截图存档，后面申诉要用。
- **不要过度热情**：买家问一个尺寸问题，你回复了 500 字还顺带介绍三个新品——这叫骚扰，不是服务。

## 动手练习

假设你收到这样一条买家消息：*"The product arrived damaged. I want a refund."*（产品到货时已损坏，我要退款。）

写一条回复，要求：
1. 在 24 小时内回复（所以现在就写）
2. 礼貌、解决问题
3. 不提评论、不引流站外
4. 包含一个明确的解决方案（退款/补发）

写完后对照本课的"红线表"自检一遍。
