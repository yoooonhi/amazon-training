# 2.2 市场需求分析

> **入门2：选品与采购**
> **本课目标**：学会用免费 + 付费工具，判断一个产品到底有没有足够的需求，值不值得做。

## 这节课解决什么问题

7.1 讲了选品 4 条标准的第 1 条是"**有没有需求**"。但新手最大的问题是：**怎么知道一个产品有没有人买？**

凭感觉是不行的。"我觉得这个会卖"——这不是分析，是赌博。你需要用**可查证的数据**来回答下面这几个问题：

- 这个关键词有多少人在搜？
- 类目里的产品卖得动吗？月销多少？
- 市场够不够大？养得起一个新品吗？
- 这个需求是全年的，还是季节性的（夏天爆、冬天死）？

这节课教你一套**从免费到付费、从粗到细**的需求分析方法，让你 30 分钟内就能判断一个品类的需求量级。

## 核心知识

### 第 1 步：搜索框联想——最便宜的需求探测

**亚马逊搜索框自带联想功能**：你在搜索栏输入一个词，它会自动弹出下拉建议。这些建议**不是随机的**，而是亚马逊根据真实用户搜索量排序的——**有人搜才会被推荐出来**。

**怎么用：**

1. 打开对应站点的 Amazon（如 amazon.com / amazon.de）
2. 在搜索框慢慢输入你的关键词（比如 `water bottle`）
3. 看下拉弹出的建议词

**判断标准：**

| 现象 | 说明 |
|---|---|
| 输入前几个字母就弹出一堆长尾词 | 需求旺盛，市场活跃 |
| 输入完整词才有少量建议 | 需求一般 |
| 输入完整词都没有建议 | 几乎没人在搜，**警告信号** |

**注意**：要用**隐身模式**（无痕窗口）查，并且最好挂对应站点的 IP——否则搜索建议会被你的浏览历史和地区污染。

### 第 2 步：BSR（Best Sellers Rank）——类目需求温度计

每个 Amazon 商品页都会显示一个 **BSR（畅销排名）**，在产品信息区的 "Best Sellers Rank" 字段。它表示这个产品在所属类目里的销量排名。

**BSR 的判断标准（粗略经验值）：**

| BSR 区间（大类目） | 含义 |
|---|---|
| 1 - 1,000 | 极热门，销量很大 |
| 1,000 - 5,000 | 需求强，是新手选品的甜区 |
| 5,000 - 20,000 | 需求中等，市场能养活产品 |
| 20,000 - 50,000 | 需求较弱，要谨慎 |
| 50,000+ | 需求很小，**通常不做** |

**怎么用：** 打开你想做的类目 Best Sellers 页面（Amazon 搜索结果页左侧栏点 "Best Sellers"），看排名前 10-20 的产品的 BSR，判断这个类目整体的需求温度。

**注意：** BSR 是**实时波动的**，看一个产品要看它最近几天的平均排名，不要被某个瞬时的数字误导。

### 第 3 步：评论数——市场成熟度信号

打开搜索结果第 1 页的 10 个产品，数它们的评论数（review count）。评论数反映市场的"成熟度"和"竞争门槛"：

| 前 10 名评论数情况 | 说明 |
|---|---|
| 全部 1000+ | 成熟市场，门槛极高，新人难进 |
| 平均 100-500，有高有低 | 健康市场，有机会 |
| 前 10 名里有几个 <100 评论 | **机会区**，新人有可能挤进去 |
| 全部 <50 | 市场很小或很新，需求存疑 |

**评论数是新手能不能进的"门槛指示器"**。前 10 名都是 1000+ 评论，意味着你想挤进去要积累的评论量很大——这部分 7.3 会详细讲。

### 第 4 步：搜索量估算——用付费工具精确测量

前三步是免费的定性判断。要精确知道**一个关键词每月被搜多少次**，得用付费工具：

| 工具 | 用途 | 备注 |
|---|---|---|
| **Helium 10**（Cerebro / Magnet） | 关键词搜索量、竞品关键词 | 主流，美国站数据最强 |
| **Jungle Scout**（关键词探测器） | 关键词搜索量、品类数据库 | 老牌，估算准确度高 |
| **SellerSprite（卖家精灵）** | 关键词、竞品、市场分析 | 中文界面，多站点支持 |

**判断标准（月搜索量，美国站量级）：**

| 月搜索量 | 含义 |
|---|---|
| 100,000+ | 大词，需求极旺，但竞争也极激烈 |
| 10,000 - 50,000 | 黄金区间，需求足、竞争可控 |
| 2,000 - 10,000 | 长尾机会，小而美 |
| <1,000 | 需求太小，单独养不起一个产品 |

**注意**：工具的搜索量是**估算值**，不同工具会有 20-30% 的差异。把它当**量级参考**（是 1 万级还是 5 万级），不要当精确数字。

### 第 5 步：市场容量——前 10 名月销量加总

光知道搜索量不够，还要看**实际卖出多少**。方法：用上面提到的工具（Jungle Scout / Helium 10 / SellerSprite）查前 10 名的预估月销量，加起来。

**判断标准：**

| 前 10 名月销总和 | 市场容量 |
|---|---|
| >3,000 件/月 | 市场够大，能养新品 |
| 1,000 - 3,000 件/月 | 中等市场，可以进 |
| 300 - 1,000 件/月 | 小市场，单品上限低 |
| <300 件/月 | 太小，**慎做** |

**简单经验值**：如果前 10 名每个都卖 300+ 件/月，市场就是合格的。

### 第 6 步：价格带分析——市场支不支持你的成本结构

光有需求不够，还要看市场**在什么价格区间成交**。打开第 1 页搜索结果，把每个产品的售价列出来，看分布：

```
例：某厨房用品关键词第 1 页售价分布
$8.99  $9.99  $10.99  $11.99  $12.99  $12.99  $14.99  $15.99  $19.99  $24.99
                ↑
        绝大多数集中在 $8-13
```

**关键判断**：

- 如果市场主流成交价是 $8-12，而你的成本结构（进货 + FBA + 佣金）**必须卖到 $15 以上**才不亏——**这个市场支不支持你**，答案就是否定的
- 新手选品的合理价格带通常是 **$15-50**：太便宜（<$15）FBA 和佣金吃光利润；太贵（>$50）买家决策成本高、广告 CPC 也贵

### 第 7 步：季节性检查——是全年需求还是季节需求

有些产品需求**全年稳定**，有些是**季节性爆发**。新手最怕选到季节性产品却没意识到——旺季备货不足、淡季库存堆积。

**怎么查：**

| 工具 | 怎么用 |
|---|---|
| **Google Trends**（trends.google.com） | 输入关键词，看 5 年搜索趋势曲线。曲线平稳 = 全年需求；明显尖峰 = 季节性 |
| **工具的 BSR 历史曲线**（Keepa / Helium 10） | 看产品 BSR 一年里的波动，BSR 大起大落 = 季节性强 |

**判断：**

| 趋势形态 | 类型 | 新手建议 |
|---|---|---|
| 一年到头平稳 | 全年需求 | **推荐**，新手友好 |
| 一年有 1-2 个明显尖峰 | 季节性（如圣诞装饰、泳衣） | 谨慎，备货时机难掌握 |
| 近 1 年持续下滑 | 需求萎缩 | **不做** |

**新手尽量选全年需求的产品**，季节性产品对节奏要求高，淡旺季备货和广告节奏都不同，容易踩坑。

## 实操要点

- **需求分析 7 步走**：搜索框 → BSR → 评论数 → 搜索量 → 市场容量 → 价格带 → 季节性。每一步都过一遍，不要跳。
- **隐身模式 + 对应站点 IP**：查搜索建议和 BSR 必须用无痕窗口，最好挂目标国家的 IP，否则数据会被污染。
- **付费工具的数字都是估算**：搜索量、月销量都有 20-50% 的误差，当量级参考可以，当精确数字会误判。
- **价格带和你的成本结构要匹配**：市场卖 $10，你成本要 $15，直接放弃，不要硬撑。
- **新手首选全年需求、$15-50 价位**：避开季节性，避开极低价和极高价。
- **每一步都要记下数字**：建一个选品表格，每个产品的 7 项数据都填进去，对比着看。

选一个你感兴趣的品类（比如"瑜伽垫"、"宠物自动喂食器"、"手冲咖啡壶"），用本课的 7 步法做一遍需求分析，把结果填进表格：

| 分析项 | 数据 | 结论 |
|---|---|---|
| 1. 搜索框联想 | （输入主词后弹出多少建议词？） | |
| 2. 前 10 名 BSR 区间 | （填范围，如 2,000-15,000） | |
| 3. 前 10 名评论数 | （填范围和平均） | |
| 4. 主关键词月搜索量 | （用任一工具估算） | |
| 5. 前 10 名月销总和 | （加总，量级即可） | |
| 6. 价格带 | （主流售价区间） | |
| 7. 季节性 | （Google Trends 看曲线，平稳/尖峰/下滑） | |

最后给一个总判断：**这个市场"需求"维度过不过关？** 如果过了，下一节 7.3 继续看"竞争"维度；如果不过关，直接淘汰，换下一个品类。
