# 6.4 Coupon与Deal

> **入门6：定价与利润**
> **本课目标**：搞清楚 Coupon（优惠券）和 Deal（秒杀/促销）的真实成本，以及怎么算"促销到底划不划算"。

## 这节课解决什么问题

新手做促销最常见的两个错觉：

1. **"打个小折无所谓"**——结果 Coupon + 促销价叠加，折扣累计，一单亏 $2，卖了 500 单，亏 $1000 还以为是"花钱买流量"。
2. **"促销能拉销量就行，利润回头再说"**——结果促销一停，销量立刻回落，等于花钱买了一次性流量，没沉淀任何东西。

问题出在：**促销不是免费的流量，它有明确的成本，而且成本不只是"折扣金额"**。这节课教你把促销当成一笔投资来算：花了多少、要换回多少单才回本、有没有叠加风险、促销完了怎么办。

## 核心知识

### Coupon 的成本结构

Coupon（优惠券）是买家主动点击领取的折扣。它的成本有两部分：

```
Coupon 总成本 = 折扣金额 × 预计领取并使用的数量 + 每次兑换的手续费
```

举个例子：你设了一个 $2 的 Coupon，预计有 100 个人会用，每次兑换亚马逊还要收 $0.60 手续费：

```
Coupon 成本 = $2 × 100 + $0.60 × 100 = $200 + $60 = $260
```

注意：**Coupon 是"领了不用不花钱，用了才花钱"**，但你设的时候要预估领取使用量。如果设太高（比如 $5 大额券），可能引来一堆只图便宜的人，成本飙升。

### Deal 的成本结构

Deal（秒杀 / 7天促销 / Prime 报名等）是一次性活动，成本结构不同：

```
Deal 总成本 = 折扣金额 × 预计销量 + 固定活动费
```

比如一个 7 天促销，打 8 折（相当于每单少赚 $2），预计卖 200 单，活动固定费 $35：

```
Deal 成本 = $2 × 200 + $35 = $400 + $35 = $435
```

固定活动费是亚马逊收的"上活动"的钱，不管你卖多卖少都要交（Lightning Deal / 7-Day Deal 都有）。

### 盈亏平衡增量销量（最关键的算法）

促销的本质是：**花钱买"本来不会买的订单"**。所以问题是——要额外多卖多少单，才能把促销成本赚回来？

```
盈亏平衡增量销量 = 促销总成本 ÷ 促销后每单的净利润
```

举例：促销总成本 $50，促销后（已经扣掉折扣）每单还能净赚 $2：

```
盈亏平衡增量销量 = $50 ÷ $2 = 25 单
```

意思是：**这次促销必须比平时多卖至少 25 单，才算不亏**。如果只多卖了 10 单，那这次促销是亏的；多卖了 40 单，才算赚。

**算这个数的时候，分母"每单净利润"必须是促销后的净利润**（已经扣了折扣），不是平时的利润。这是新手容易算错的地方。

### 叠加风险：折扣会累加，可能亏到负数

亚马逊的折扣机制里，**多种促销可能叠加**——Coupon、Sale Price（促销价）、Prime 折扣等可以同时生效，折扣是累加的，不是取最大的那个。

最危险的是 **Coupon + Sale Price 叠加**：

```
原价 $11.99
设了 Sale Price $9.99（立减 $2）
又挂了一个 $2 Coupon
买家实付 = $9.99 − $2 = $7.99
实际折扣 = $11.99 − $7.99 = $4（折扣率 33%）
```

如果 $7.99 已经低于你的成本+FBA+佣金，**每一单都在亏钱**，卖得越多亏得越多。

**铁律**：设任何新促销之前，先查这个产品**现在挂了哪些促销**（Coupon、Sale Price、Prime 折扣、活动报名），确认不会叠加出负利润。

### 2026 价格抓取机制（秒杀前必读）

> 这一小节是 2026 年新增的"隐藏成本"——促销会污染你未来 90 天的价格基准，让你下一次促销的价格越压越低。

### 四个抓取规则

**规则 1：BD（Best Deal）时长改成 1-14 天灵活窗口**

非旺季可以自主设置 BD 时长（1-14 天任选）。BD 相邻活动间隔延长到 **21 天**，BD 和 LD 建议间隔 **30 天**（最低 21 天）。

含义：以前 BD 只能跑固定时长，现在灵活了，但相邻间隔拉长了，意味着"连场秒杀"没那么容易做了。

**规则 2：Coupon 长期挂会拉低你的价格基准**

如果你 90 天内有 70% 以上时间都挂着 Coupon 或促销价，系统会判定这个低价就是你的"常规售价"，**下次报秒杀时，基准价被压低**，秒杀价必须在此基础上再降。

举例：原价 $11.99，长期挂 $2 Coupon。90 天后系统认为你的"常规价"是 $9.99。下次报 LD，秒杀价要从 $9.99 再降 10-15%，变成 $8.5 左右——利润被越压越薄。

**规则 3：Prime Day / 大促的折后价会被抓取**

PD、黑五这种大促的价格，会被纳入后续 90 天的"历史最低价"参考。也就是说，**大促打折的代价不只是当次少赚，还会污染你接下来 3 个月的秒杀价**。

**规则 4：划线价（List Price）大促前 2 周冻结**

2026 年 Prime Day 起，大促前**提前两周**强制冻结 List Price 修改。想在大促前临时抬高划线价制造"大折扣"的玩法失效了。

::: warning 行业共识，非官方原文
以上四个抓取规则中，规则 2-4 的具体细节（90 天 70% 阈值、PD 抓取范围、划线价冻结 2 周）**主要来自第三方解读和卖家社区共识**，亚马逊后台没有单一官方公告直接表述。授课时按"行业共识规则，以卖家后台实际提报提示为准"理解。规则 1（1-14 天 BD）有亚马逊全球开店官方出处。
:::

### 秒杀前 30 天"价格养干净"SOP

为了不让历史促销污染秒杀价，秒杀前 30 天要做这几件事：

| 时间 | 动作 |
|---|---|
| 秒杀前 30 天 | **停掉所有 Coupon 和其他促销**，让价格回到"干净"的常规售价 |
| 秒杀前 14 天 | **锁定 List Price（划线价）不要再改**，避免触发冻结或审核 |
| 秒杀前 7 天 | **不要做 PD / 黑五这类大促**，避免折后价被抓取污染基准 |
| 秒杀当天 | 用盈亏平衡增量销量公式（见上节）算清楚这次到底赚不赚 |
| 秒杀后 | 把"下次秒杀价会被压低多少"作为隐性成本计入这次 Deal 的总成本 |

> 一句话总结：**促销不是孤立的一次活动，它会污染你未来 90 天的价格基准。** 算 Deal 利润时，要把"被压低的下次秒杀价"当作隐性成本一起算进去，不然你以为赚了，其实把未来的利润透支了。

### 促销后的"收尾计划"

促销不是"卖完就结束"。一个负责任的促销，在开始之前就要想好三件事：

1. **价格怎么恢复**：促销价结束后，是直接回原价（可能销量断崖），还是分步回调（$9.99 → $10.99 → $11.99）？
2. **广告预算怎么调**：促销期间广告通常会加码，结束后要恢复到正常水平，别让广告继续按促销强度烧。
3. **自然订单能不能撑住**：促销带来的销量里，有多少转化成了"自然搜索能持续出单"的排名提升？如果促销一停销量立刻归零，说明这次促销只买到了一次性流量，没沉淀。

## 实操要点

- **每个促销都先算"盈亏平衡增量销量"**：总成本 ÷ 促销后单件利润。算不出这个数，就别上促销。
- **设促销前必查叠加**：进 Listing 看现在挂了哪些折扣，Coupon + Sale Price 叠加是头号杀手。
- **促销后的单件利润必须是促销后的**（已扣折扣），不要拿平时的利润算分母。
- **促销前就写好收尾计划**：价格怎么回、广告怎么收、怎么判断有没有沉淀。
- **大额 Coupon 慎用**：$2 以上的 Coupon 容易引来"只图便宜"的买家，促销一停就消失，没沉淀。

## 动手练习

你有一个产品，平时售价 $11.99，平时每单净利润 $2.50。现在你想做一个 7 天促销，促销价 $9.99，预计 7 天能卖 150 单（平时 7 天卖 80 单），活动固定费 $35。

请算：
1. 每单折扣是多少？（$11.99 − $9.99）
2. 促销总成本 = 折扣 × 预计销量 + 固定费 = ？
3. 促销后每单净利润是多少？（平时 $2.50 − 每单折扣）
4. 盈亏平衡增量销量 = 促销总成本 ÷ 促销后单件利润 = ？
5. 预计增量销量 = 150 − 80 = 70 单，比盈亏平衡需要的多还是少？这次促销划不划算？

如果算出来盈亏平衡要 50 单、你预计多卖 70 单，那这次促销是赚的；如果反过来，就别做。
